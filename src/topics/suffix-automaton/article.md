# 后缀自动机 (Suffix Automaton)

## 概念解释

后缀自动机（Suffix Automaton，简称 SAM）是一种极其高效的**字符串数据结构**，它是一个**有向无环图（DAG）**，能够接受原字符串的**所有后缀**，同时也能接受所有子串。

### 核心定义

给定字符串 `S`，它的后缀自动机满足：

- **初始状态**：从初始状态出发，沿着转移边走，能且仅能走到 S 的所有子串
- **接受状态**：从初始状态到任意接受状态的路径上，字符拼接起来恰好是 S 的某个后缀
- **状态含义**：每个状态代表一个 **endpos 等价类**

### 什么是 endpos？

对于子串 `t`，`endpos(t)` 定义为 t 在字符串 S 中所有出现位置的**结束位置集合**。

例如，对于字符串 `S = "abcbc"`：

```
子串     出现位置        endpos
"bc"     [1,2] [3,4]    {2, 4}
"abc"    [0,2]          {2}
"bcbc"   [1,4]          {4}
"a"      [0,0]          {0}
"c"      [2,2] [4,4]    {2, 4}
```

可以看到，`"bc"` 和 `"c"` 的 endpos 集合都是 `{2, 4}`，它们属于同一个 **endpos 等价类**，在 SAM 中对应同一个状态。

### 状态的含义

每个状态 v 对应一组子串，这些子串的 endpos 集合相同。其中：
- **最长子串**长度为 `len(v)`
- **最短子串**长度为 `len(link(v)) + 1`
- 该状态包含的所有子串长度连续，且是其最长子串的连续后缀

### 基本术语

| 术语 | 说明 |
|------|------|
| 状态 (State) | SAM 中的节点，代表一个 endpos 等价类 |
| 转移 (Transition) | 状态之间的有向边，标记一个字符 |
| suffix link | 从当前状态指向其 endpos 严格超集的状态 |
| len(v) | 状态 v 对应的最长子串长度 |
| link(v) | 状态 v 的 suffix link 指向的状态 |

## 为什么重要

后缀自动机在字符串算法中有着独特的优势：

1. **线性空间**：O(n) 个状态和 O(n) 条转移边，空间常数比后缀树更小
2. **在线构建**：逐字符添加，每添加一个字符即可更新自动机
3. **功能强大**：一棵自动机可以解决大量字符串问题
4. **实现简洁**：相比 Ukkonen 构建后缀树，SAM 的代码更短更直观

### 可以解决的经典问题

| 问题 | 时间复杂度 |
|------|-----------|
| 子串判断 | O(m)，m 为模式串长度 |
| 不同子串个数 | O(n) |
| 子串出现次数 | O(n) 预处理 + O(m) 查询 |
| 最长公共子串 | O(n + m) |
| 字典序第 k 小子串 | O(n + k) |
| 最短非子串 | O(n) |
| 最长公共子串（多串） | O(sum of lengths) |

### 与后缀树的对比

| 特性 | 后缀自动机 (SAM) | 后缀树 |
|------|-----------------|--------|
| 空间复杂度 | O(n)，常数小 | O(n)，常数较大 |
| 构建时间 | O(n) | O(n)（Ukkonen） |
| 实现难度 | 中等 | 较高 |
| 数据结构 | 有向无环图 | 树 |
| 核心操作 | 在线逐字符添加 | 在线逐字符添加 |
| 子串查询 | O(m) | O(m) |
| 缓存友好 | 较好 | 一般 |

## 核心原理

### SAM 的两个核心性质

**性质一：后缀链接构成一棵树**

所有 suffix link 指向初始状态的边构成一棵以初始状态为根的树。这棵树的结构反映了 endpos 集合之间的包含关系。

```
         root (初始状态)
        / | \
       /  |  \
      v1  v2  v3
     /
    v4
```

**性质二：转移构成 DAG**

所有转移边构成一个有向无环图。从初始状态出发，沿着转移走，能到达的路径对应原字符串的子串。

### 在线构建过程

SAM 最精妙的地方在于它支持**在线构建**：逐个添加字符，每次添加的摊还代价为 O(1)。

#### 算法概述

假设当前已经构建了 S[0..i-1] 的 SAM，现在要添加字符 S[i]：

1. 创建新状态 `cur`，`len(cur) = len(last) + 1`
2. 从 `last`（上一个字符对应的状态）沿 suffix link 回溯：
   - 对于回溯路径上的每个状态 p，如果 p 没有通过 S[i] 的转移，则添加 `p --S[i]--> cur`
3. 沿 suffix link 回溯到某个状态 p 时，如果 p 已经有通过 S[i] 的转移指向 q：
   - **情况一**：如果 `len(p) + 1 == len(q)`，直接设置 `link(cur) = q`
   - **情况二**：否则需要**分裂** q，创建 clone 节点

#### 克隆节点（Clone）

当遇到情况二时，需要创建 q 的克隆节点 `clone`：
- `len(clone) = len(p) + 1`
- 复制 q 的所有转移和 suffix link
- 将 q 和 cur 的 suffix link 指向 clone
- 将原来指向 q 的部分转移重定向到 clone

```
分裂前:                    分裂后:
  p --c--> q               p --c--> clone --c--> (q的子树)
                            q = clone + cur 的 suffix link 指向 clone
```

### 伪代码

```
function extend(c):
    cur = new_state(len[last] + 1)
    p = last
    while p != -1 and not has_transition(p, c):
        transition[p][c] = cur
        p = link[p]
    if p == -1:
        link[cur] = 0  // 指向初始状态
    else:
        q = transition[p][c]
        if len[p] + 1 == len[q]:
            link[cur] = q
        else:
            clone = new_state(len[p] + 1)
            transition[clone] = transition[q]  // 复制转移
            link[clone] = link[q]
            while p != -1 and transition[p][c] == q:
                transition[p][c] = clone
                p = link[p]
            link[q] = link[cur] = clone
    last = cur
```

### 构建示例

以字符串 `"abcbc"` 为例：

```
添加 'a':  state 1, len=1, link=0
添加 'b':  state 2, len=2, link=0
添加 'c':  state 3, len=3, link=0
添加 'b':  state 4, len=4, link=2
添加 'c':  state 5, len=5, link=3
```

### 不同子串个数

每个状态 v 贡献的独有子串数量为：

```
contribution(v) = len(v) - len(link(v))
```

总不同子串数 = 所有状态的贡献之和。

### 子串出现次数

通过 suffix link 树上的拓扑排序，从叶子向根累加：

1. 每个状态初始出现次数 = 1（如果是 endpos 中的位置）或 0
2. 沿 suffix link 树自底向上累加
3. 状态 v 的出现次数 = 其子树中所有叶子的出现次数之和

## 常见错误

### 1. 混淆后缀自动机和后缀树

```typescript
// 后缀自动机：有向无环图，每个状态代表 endpos 等价类
// 后缀树：树形结构，每条边上存储子字符串

// SAM 的每个状态可能对应多个子串
// 后缀树的每个节点对应唯一的路径字符串
```

后缀自动机通过 endpos 等价类压缩信息，而后缀树通过边压缩存储。两者虽然都能解决类似的字符串问题，但数据结构和构建过程完全不同。

### 2. clone 节点处理错误

```typescript
// 错误：忘记复制 q 的所有转移
clone.transition = {}  // 空的！应该复制 q 的转移

// 正确：
clone.transition = { ...q.transition }  // 复制所有转移
clone.link = q.link                      // 复制 suffix link

// 错误：忘记将部分转移重定向到 clone
// 应该沿 suffix link 回溯，将原来指向 q 的转移改为指向 clone
while (p !== -1 && p.transition[c] === q) {
  p.transition[c] = clone
  p = p.link
}
```

clone 节点的处理是 SAM 构建中最容易出错的部分。关键是：clone 继承 q 的转移和 link，然后需要将回溯路径上指向 q 的转移重定向到 clone。

### 3. link 指针方向搞反

```typescript
// 错误理解：link[v] 指向 v 的前缀
// 正确理解：link[v] 指向的状态，其 endpos 集合是 v 的 endpos 集合的严格超集

// suffix link 的方向：从较长子串指向较短子串
// 例如：状态 v 对应 "abc"，link(v) 对应 "bc"
// "bc" 的 endpos 是 "abc" 的 endpos 的超集
```

suffix link 的方向是从"长子串"指向"短子串"（去掉第一个字符）。在 suffix link 树上，根是初始状态（对应空串），叶子是 len 最大的状态。

### 4. 忘记初始化

```typescript
// 错误：没有正确初始化初始状态
// 初始状态（状态 0）：len = 0, link = -1

// 正确：
states[0] = { len: 0, link: -1, transitions: {} }
last = 0
```

### 5. len 值设置错误

```typescript
// 错误：clone 的 len 设置为 len[q]
// 正确：clone 的 len 应该是 len[p] + 1

// clone 的作用是：将 q 中长度 <= len[p] + 1 的子串分离出来
// 所以 clone.len = len[p] + 1
```

## 实际应用

### 1. 子串出现次数统计

```typescript
// 构建 SAM 后，通过 suffix link 树计算每个状态的出现次数
// 状态 v 的出现次数 = 其在 suffix link 树中的子树大小

// 查询模式串 P 的出现次数：
// 在 SAM 上沿 P 的字符走转移，到达状态 v
// v 的出现次数即为答案
```

### 2. 最长公共子串

```typescript
// 求字符串 S1 和 S2 的最长公共子串：
// 1. 构建 S1 的 SAM
// 2. 在 SAM 上匹配 S2 的每个字符
// 3. 记录匹配过程中到达的最大 len 值

// 时间复杂度：O(|S1| + |S2|)
```

### 3. 不同子串个数

```typescript
// 构建 SAM 后，不同子串个数 = 所有状态的 (len[v] - len[link[v]]) 之和
// 这是因为每个状态 v 贡献了恰好 (len[v] - len[link[v]]) 个独有子串
```

### 4. 字典序第 k 小子串

```typescript
// 构建 SAM 后，利用 suffix link 树上的 DP：
// 1. 计算每个状态可达的不同子串数 dp[v]
// 2. 从初始状态出发，按字典序遍历转移边
// 3. 根据 dp 值判断第 k 小子串在哪条转移边上
```

### 5. 最短非子串

```typescript
// 找到字符串 S 中最短的、不是 S 子串的字符串
// 在 SAM 上 BFS，找到第一个不存在的转移即可
// 时间复杂度：O(|S|)
```

## 总结

后缀自动机是字符串算法中极为精妙的数据结构：

**核心优势**：
- O(n) 线性空间，常数小
- O(n) 在线构建，摊还 O(1) 添加每个字符
- 一棵自动机解决大量字符串问题
- 代码实现相对简洁

**适用场景**：
- 需要统计子串出现次数
- 求不同子串个数
- 求最长公共子串
- 字典序相关查询
- 各种子串计数和查询问题

**学习建议**：
- 先理解 endpos 等价类的概念
- 手动模拟小规模字符串的 SAM 构建过程
- 重点理解 clone 节点的创建条件和作用
- 通过可视化工具辅助理解
- 实现代码后用简单例子验证正确性

后缀自动机的学习曲线较陡，但一旦理解其核心思想，它是解决字符串问题最优雅的工具之一。
