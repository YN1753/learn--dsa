# 数论变换 NTT (Number Theoretic Transform)

## 概念解释

数论变换（NTT）是**快速傅里叶变换（FFT）在有限域上的推广**。它用模运算中的**原根**代替复数中的**单位根**，从而在整数模意义下完成多项式乘法。

### 核心思想

FFT 的核心是利用复数单位根 $\omega_n$ 的性质来分治计算多项式乘法。NTT 将这一思想搬到模 $p$ 的有限域中：

- 用模 $p$ 下的**原根** $g$ 生成单位根
- 用 $g_n = g^{(p-1)/n} \pmod{p}$ 代替 $\omega_n = e^{2\pi i/n}$
- 所有运算变为整数模运算，无浮点误差

### 关键定义

| 术语 | 说明 |
|------|------|
| 原根 (Primitive Root) | 模 $p$ 下阶为 $p-1$ 的元素，其幂次可生成所有非零元素 |
| NTT友好素数 | 可表示为 $k \cdot 2^m + 1$ 的素数，保证足够大的变换长度 |
| 蝶形运算 | NTT/FFT 的基本运算单元，进行加减和乘法组合 |
| 位逆序置换 | 将下标按二进制位翻转重新排列 |

## 为什么重要

NTT 在算法竞赛和工程中极其重要：

1. **无精度损失**：所有计算都是整数模运算，避免 FFT 的浮点误差累积
2. **高效多项式乘法**：$O(n \log n)$ 时间完成两个 $n$ 次多项式的乘法
3. **竞赛必备**：大量题目需要精确的多项式/卷积运算，NTT 是首选工具
4. **扩展性强**：可以实现多项式求逆、开根、对数、指数等高级操作
5. **应用广泛**：卷积、字符串匹配、组合计数、生成函数等

## 核心原理

### 1. 原根与单位根

对于素数 $p$，若 $g$ 是模 $p$ 的原根，则 $g$ 的阶为 $p-1$，即 $g^{p-1} \equiv 1 \pmod{p}$，且 $g^0, g^1, \ldots, g^{p-2}$ 恰好遍历 $1$ 到 $p-1$。

NTT 需要的 $n$ 次单位根定义为：

$$g_n = g^{(p-1)/n} \pmod{p}$$

这要求 $n$ 是 $p-1$ 的因子。

### 2. 常用模数

| 模数 $p$ | 形式 | 最大变换长度 | 原根 |
|----------|------|-------------|------|
| 998244353 | $119 \times 2^{23} + 1$ | $2^{23}$ | 3 |
| 1004535809 | $479 \times 2^{21} + 1$ | $2^{21}$ | 3 |
| 469762049 | $7 \times 2^{26} + 1$ | $2^{26}$ | 3 |

**998244353** 是最常用的NTT模数，因为：
- 它是质数
- $p - 1 = 2^{23} \times 119$，支持长达 $2^{23} \approx 8 \times 10^6$ 的变换
- 约 $10^9$ 量级，两个结果相乘不会溢出64位整数
- 原根是 3，计算方便

### 3. 蝶形运算

NTT 的蝶形运算与 FFT 完全类似，只是用模运算替代复数运算：

```
设当前处理长度为 len，单位根为 w
对于每对 (i, i + len/2)：
  u = a[i]
  v = a[i + len/2] * w % p
  a[i]         = (u + v) % p
  a[i + len/2] = (u - v + p) % p
```

### 4. 算法流程

```
NTT(a[], n, flag):
  1. 位逆序置换 (Bit-reversal Permutation)
  2. 对于 len = 2, 4, 8, ..., n:
     计算当前层单位根 w = g^((p-1)/len) (若flag=-1则取逆)
     对于每个蝶形块:
       执行蝶形运算
  3. 如果 flag = -1 (逆变换):
     每个元素除以 n (乘以 n 的模逆元)
```

### 5. 多项式乘法完整流程

```
给定多项式 A(x) 度为 n-1，B(x) 度为 m-1

1. 将长度扩展到 >= n+m-1 的 2 的幂次 N
2. 补零到长度 N
3. NTT(A, N, 1)  -- 正变换
4. NTT(B, N, 1)  -- 正变换
5. 逐点相乘: C[i] = A[i] * B[i] % p
6. NTT(C, N, -1) -- 逆变换
7. C 即为结果
```

## 与 FFT 的比较

| 特性 | FFT | NTT |
|------|-----|-----|
| 数域 | 复数 | 有限域 $\mathbb{Z}_p$ |
| 精度 | 浮点，有误差 | 精确整数运算 |
| 单位根 | $e^{2\pi i/n}$ | $g^{(p-1)/n} \bmod p$ |
| 时间复杂度 | $O(n \log n)$ | $O(n \log n)$ |
| 变换长度限制 | 任意2的幂 | 受模数限制 |
| 适用场景 | 信号处理、一般计算 | 竞赛、精确卷积 |
| 实现复杂度 | 需要复数运算 | 需要模运算和快速幂 |

### NTT 的优势
- 无浮点误差，结果精确
- 整数运算通常比复数运算快
- 不需要处理精度问题

### NTT 的劣势
- 变换长度受模数限制
- 需要选择合适的模数
- 可能需要三模NTT（CRT合并）来扩大范围

## 常见错误

### 1. 忘记取模

```typescript
// 错误：中间结果溢出
const mul = (a: number, b: number, p: number) => a * b % p

// 正确：使用 BigInt 或确保不溢出
const mul = (a: number, b: number, p: number) => {
  return Number(BigInt(a) * BigInt(b) % BigInt(p))
}
```

### 2. 位逆序置换错误

```typescript
// 错误：没有正确计算位逆序
for (let i = 1; i < n; i++) {
  const j = reverseBits(i) // 需要正确实现
  if (i < j) swap(a, i, j)
}
```

### 3. 逆变换忘记除以 n

```typescript
// 逆变换后需要乘以 n 的模逆元
const invN = modPow(n, p - 2, p) // 费马小定理
for (let i = 0; i < n; i++) {
  a[i] = a[i] * invN % p
}
```

### 4. 变换长度不够

```typescript
// 错误：没有扩展到足够大的2的幂次
const n = a.length + b.length - 1
const N = 1 << Math.ceil(Math.log2(n)) // 必须是2的幂
```

## 实际应用

### 1. 大整数乘法

将大整数看作多项式，系数为每一位的数字，用 NTT 计算乘积后处理进位。

### 2. 字符串匹配

使用卷积进行模式匹配：将模式串和文本串编码为多项式，通过 NTT 计算卷积来快速匹配。

### 3. 组合计数

许多组合计数问题可以转化为多项式乘法，如：
- 划分数（生成函数）
- 卡特兰数
- 斯特林数

### 4. 生成函数运算

NTT 支持生成函数的各种运算：
- 乘法：对应序列的卷积
- 求逆：$B(x) \equiv A(x)^{-1} \pmod{x^n}$
- 开根：$B(x)^2 \equiv A(x) \pmod{x^n}$
- 对数/指数：$B(x) \equiv \ln A(x)$ 或 $B(x) \equiv e^{A(x)}$

### 5. 概率与期望计算

某些概率问题可以转化为多项式运算，利用 NTT 加速计算。

## 总结

NTT 是算法竞赛和数学计算中的重要工具：

**核心优势**：
- 精确整数运算，无浮点误差
- $O(n \log n)$ 时间复杂度
- 实现相对简洁

**适用场景**：
- 需要精确结果的多项式乘法
- 竞赛编程中的卷积问题
- 大整数乘法
- 生成函数相关计算

**注意事项**：
- 选择合适的模数（如 998244353）
- 注意变换长度限制
- 中间结果可能需要使用 BigInt 防止溢出

掌握 NTT 是深入学习多项式算法和高级数学竞赛题目的重要基础。
