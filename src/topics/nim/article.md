# Nim 游戏

## 概念解释

Nim 游戏是组合博弈论中最经典的博弈游戏，也是理解其他博弈问题的基础。

### 游戏规则

1. 有 **n 堆石子**，每堆有若干个石子
2. 两名玩家**轮流**操作
3. 每次操作可以从**任意一堆**中取走**至少一个**石子（可以取完）
4. **取走最后一个石子的人获胜**（正常玩法）

### 一个简单例子

假设游戏有三堆石子：`[3, 4, 5]`

```
堆 1: ●●●
堆 2: ●●●●
堆 3: ●●●●●
```

玩家 A 先手，从第三堆取走 2 个石子，局面变为 `[3, 4, 3]`。

玩家 B 从第二堆取走 1 个石子，局面变为 `[3, 3, 3]`。

...以此类推，直到某位玩家取走最后一个石子获胜。

### 胜负判定的核心

看起来这个游戏非常复杂——需要分析每一种可能的操作。但数学家们发现了一个惊人的结论：

> **将所有堆的石子数量进行异或（XOR）运算，结果为 0 则先手必败，非 0 则先手必胜。**

这就是著名的 **Bouton 定理**，由 Charles Bouton 在 1901 年证明。

## 为什么重要

Nim 游戏在计算机科学和博弈论中具有核心地位：

1. **博弈论的基石**：几乎所有组合博弈问题都可以归结为 Nim 游戏或其变种
2. **SG 定理的特例**：Nim 游戏是 Sprague-Grundy 定理最重要的特例和直觉来源
3. **异或运算的深刻含义**：揭示了二进制运算在博弈分析中的神奇力量
4. **竞赛高频考点**：算法竞赛中博弈论题目的核心基础
5. **游戏 AI 基础**：许多游戏 AI 的底层逻辑都基于 Nim 游戏的思想

## 核心原理

### 异或（XOR）运算回顾

异或运算是按位进行的二进制运算：

```
0 XOR 0 = 0
0 XOR 1 = 1
1 XOR 0 = 1
1 XOR 1 = 0
```

关键性质：
- **自反性**：a XOR a = 0
- **交换律**：a XOR b = b XOR a
- **结合律**：(a XOR b) XOR c = a XOR (b XOR c)
- **单位元**：a XOR 0 = a

### Nim 游戏的异或定理

**定理**：设 n 堆石子的数量分别为 a1, a2, ..., an，令 S = a1 XOR a2 XOR ... XOR an。

- 若 S = 0，则当前局面为 **P 态**（先手必败）
- 若 S ≠ 0，则当前局面为 **N 态**（先手必胜）

### 为什么异或能判定胜负？

**直观理解**：

当 S ≠ 0 时，先手总能找到一步操作，使得操作后 S' = 0。

具体做法：设 S 的最高位为第 k 位，则至少存在一堆 ai，其第 k 位为 1。将 ai 变为 ai XOR S（一定比 ai 小），则新的异或值变为 S XOR ai XOR (ai XOR S) = 0。

当 S = 0 时，无论先手怎么取，都会使某堆数量减少，导致新的异或值 S' ≠ 0。

因此：**S = 0 的局面只能走到 S ≠ 0 的局面，而 S ≠ 0 的局面总能走到 S = 0 的局面。**

游戏最终会到达所有堆都为空（S = 0），此时上一个操作的人（后手）获胜——所以 S = 0 时先手必败。

### 计算示例

以 `[3, 4, 5]` 为例：

```
3 = 011 (二进制)
4 = 100 (二进制)
5 = 101 (二进制)

异或计算：
  011
  100
  ---
  111   (3 XOR 4 = 7)

  111
  101
  ---
  010   (7 XOR 5 = 2)

S = 2 ≠ 0 → N 态（先手必胜）
```

先手必胜策略：找到一堆 ai，使得 ai XOR S < ai。

- 堆 1: 3 XOR 2 = 1 < 3，可取走 2 个，变为 [1, 4, 5]，异或 = 0
- 堆 3: 5 XOR 2 = 7 > 5，不行
- 堆 2: 4 XOR 2 = 6 > 4，不行

所以先手应从第一堆取走 2 个石子。

### 反常玩法（Misere Nim）

在反常玩法中，**取走最后一个石子的人失败**。

结论几乎相同：当只剩下一堆超过 1 个石子的堆时，将策略从"使异或为 0"变为"使异或非 0"。其他情况策略不变。

## 常见错误

### 1. 误以为堆数多就复杂

```
错误直觉：堆越多越难分析

正确理解：无论多少堆，只需计算一次异或即可判定胜负
异或的时间复杂度为 O(n)，n 为堆数
```

### 2. 混淆 XOR 和 OR/AND

```
错误：用 OR 运算判断（如 3 OR 4 OR 5 = 7 ≠ 0，判断为必胜——碰巧对了但方法错误）
错误：用 AND 运算判断

正确：必须使用 XOR（异或）运算
3 AND 4 AND 5 = 0  ← 用 AND 会误判为必败！
3 XOR 4 XOR 5 = 2  ← 正确的异或值
```

### 3. 忘记"至少取一个"的约束

```
错误操作：不取石子，让对手面对同一局面

正确规则：每次必须取走至少一个石子
```

### 4. 取石子数量超过堆中实际数量

```
错误：从只有 3 个石子的堆中取走 5 个

正确：取走的数量不能超过该堆的石子数量
```

### 5. 误以为必胜态一定能一步获胜

```
错误：先手必胜 = 马上获胜

正确：先手必胜意味着存在一种策略，保证最终获胜
可能需要多步操作才能到达胜利
```

## 实际应用

### 1. 算法竞赛博弈题

```typescript
// Nim 游戏核心判定
function nimWinning(piles: number[]): boolean {
  let xor = 0
  for (const pile of piles) {
    xor ^= pile
  }
  return xor !== 0  // true = 先手必胜
}

// 找出最优操作
function nimBestMove(piles: number[]): [number, number] | null {
  let xor = 0
  for (const pile of piles) xor ^= pile
  if (xor === 0) return null  // 必败态，无必胜策略

  for (let i = 0; i < piles.length; i++) {
    const target = piles[i] ^ xor  // 目标数量
    if (target < piles[i]) {
      return [i, piles[i] - target]  // [堆索引, 取走数量]
    }
  }
  return null
}
```

### 2. 游戏 AI 设计

许多棋类和策略游戏的 AI 都借鉴了 Nim 游戏的思想：

- 计算当前局面的"异或值"
- 选择使对手面对异或值为 0 的操作
- 在无法获胜时选择最"均衡"的操作

### 3. 密码学

Nim 游戏的数学结构与某些密码学算法有深层联系，尤其是涉及 XOR 运算的部分。

### 4. 分布式系统

某些分布式算法使用类似 Nim 的博弈思想来实现公平调度和资源分配。

## 总结

Nim 游戏是组合博弈论的起点：

**核心定理**：n 堆石子数量异或为 0 则先手必败，否则先手必胜。

**关键洞察**：
- 异或为 0 的局面只能走到异或非 0 的局面
- 异或非 0 的局面总能走到异或为 0 的局面
- 最终游戏到达空状态（异或 = 0），最后操作者获胜

**重要性质**：
- 时间复杂度 O(n) 即可判定胜负
- 可以在 O(n) 内找到最优操作
- 是 Sprague-Grundy 定理的特例和基础

**学习路径**：
- 先理解经典 Nim 的异或判定法
- 再学习 SG 函数和 SG 定理
- 最后掌握各种博弈变种的分析方法

掌握 Nim 游戏，就掌握了分析一类博弈问题的核心工具。
