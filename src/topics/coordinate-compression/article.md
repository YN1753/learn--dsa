# 坐标压缩 (Coordinate Compression)

## 概念解释

**坐标压缩**（也称离散化）是一种预处理技巧，用于将**大范围、稀疏的离散值**映射为**连续的小范围整数索引**。

简单来说：

- 你有一组数据，值可能非常大（例如 10^9），或者分布非常稀疏
- 但实际出现的**不同值**数量相对较少（例如只有 10^5 个）
- 坐标压缩将这些值按大小排序、去重，然后分配 0、1、2、... 的连续索引

**映射过程：**

```
原始值:   [100, 5000, 300, 100, 5000]
去重排序: [100, 300, 5000]
映射:     100 → 0,  300 → 1,  5000 → 2
结果:     [0, 2, 1, 0, 2]
```

压缩后，原本跨度巨大的值域被"压缩"到了一个紧凑的连续区间内，使得基于数组的数据结构（线段树、树状数组、前缀和等）可以高效使用。

## 为什么重要

在算法竞赛和实际开发中，坐标压缩解决了一个常见的困境：

1. **内存瓶颈**：线段树通常需要 4 倍于值域大小的空间。当值域达到 10^9 时，直接开数组需要约 4GB 内存，完全不可行。坐标压缩后，只需 4 * (不同值个数) 的空间。

2. **算法可行性**：许多经典算法（离线处理、扫描线、事件排序等）依赖于将坐标映射到连续索引才能正确工作。

3. **通用性**：坐标压缩不仅适用于整数，也适用于浮点数、字符串等任何可以比较大小的类型。

4. **简单高效**：实现只需要排序和二分查找，时间复杂度 O(n log n)，代码量极小。

## 核心原理

### 压缩步骤

坐标压缩的核心步骤非常简单：

**步骤 1：收集所有出现的值**

将所有需要压缩的值收集到一个数组中。

**步骤 2：去重并排序**

```
去重: 使用 Set 或排序后跳过重复元素
排序: 按升序排列
```

**步骤 3：建立映射**

为每个值分配其在排序后数组中的索引（0-indexed）。

```
sorted[i] → i
```

**伪代码：**

```
function compress(values):
    sorted = unique(values) sorted ascending
    map = {}
    for i = 0 to sorted.length - 1:
        map[sorted[i]] = i
    return sorted, map
```

**时间复杂度**：O(n log n) — 主要来自排序
**空间复杂度**：O(n) — 存储去重后的值和映射表

### 查询与反向映射

- **正向映射**（值 → 索引）：通过哈希表 O(1) 查询
- **反向映射**（索引 → 值）：直接查排序后的数组 sorted[index]
- **存在性判断**：通过二分查找判断某个值是否在原始数据中出现

### 二分查找实现

如果没有使用哈希表，也可以用二分查找来获取压缩索引：

```
function getIndex(sorted, value):
    // 找到 value 在 sorted 中的位置
    // 即 lower_bound: 第一个 >= value 的位置
    lo = 0, hi = sorted.length
    while lo < hi:
        mid = (lo + hi) / 2
        if sorted[mid] < value:
            lo = mid + 1
        else:
            hi = mid
    return lo  // 若 sorted[lo] == value 则找到，否则 value 不存在
```

## 常见错误

### 1. 忘记去重

```typescript
// 错误：不去重直接排序，导致映射索引不连续
function wrongCompress(values: number[]): number[] {
  const sorted = [...values].sort((a, b) => a - b)
  // 如果 values = [100, 100, 300]，
  // sorted = [100, 100, 300]
  // 100 映射到 0 和 1，浪费空间且索引不正确
}

// 正确：先去重再排序
function correctCompress(values: number[]): Map<number, number> {
  const sorted = [...new Set(values)].sort((a, b) => a - b)
  const map = new Map<number, number>()
  sorted.forEach((v, i) => map.set(v, i))
  return map
}
```

### 2. 索引从 1 开始还是从 0 开始

```typescript
// 根据使用场景选择：
// 线段树通常偏好 1-indexed
sorted.forEach((v, i) => map.set(v, i + 1))  // 1-indexed

// 一般的数组操作偏好 0-indexed
sorted.forEach((v, i) => map.set(v, i))  // 0-indexed

// 关键：保持一致！不要混用
```

### 3. 处理区间端点的离散化

```typescript
// 错误：只压缩端点值，忽略区间内的点
// 区间 [1, 100] 和 [50, 200] 在压缩后可能变成相邻索引
// 但实际上它们有重叠区域 [50, 100]

// 正确（离线算法中）：考虑是否需要在相邻值之间插入"间隔点"
// 例如：对于区间覆盖问题，有时需要将相邻两个不同值之间的"间隙"也作为一个索引
```

### 4. 浮点数精度问题

```typescript
// 错误：直接比较浮点数
const values = [0.1 + 0.2, 0.3]  // 0.1 + 0.2 !== 0.3 in floating point!

// 正确：使用容差比较，或在压缩前将浮点数转为整数
const values = [Math.round((0.1 + 0.2) * 1e9), Math.round(0.3 * 1e9)]
```

## 实际应用

### 1. 配合线段树 / 树状数组

这是坐标压缩最常见的应用场景。当需要在大范围坐标上进行区间操作（区间加、区间查询等）时：

```
原始坐标: [1, 10^9, 5, 10^9-1]
压缩后:   [0, 3, 1, 2]
线段树:   只需 4 * 4 = 16 个节点（而非 4 * 10^9）
```

### 2. 离线算法

许多离线算法（如莫队算法、CDQ分治）需要将所有查询一起处理，坐标压缩是预处理的标准步骤。

### 3. 扫描线算法

在计算几何的扫描线算法中，需要将 x 坐标或 y 坐标压缩后，才能在线段树上进行区间覆盖/查询操作。

### 4. 事件排序与处理

在模拟或事件驱动的算法中，将时间点或事件坐标压缩后，可以用数组直接索引，避免使用哈希表的常数开销。

### 5. 图论中的节点映射

当图的节点编号不连续或范围很大时，坐标压缩可以将节点映射到 0~n-1 的连续编号，方便使用邻接矩阵或数组存储。

## 总结

坐标压缩是一种简单而强大的预处理技巧：

**核心思想**：
- 排序 + 去重 + 分配索引
- 将稀疏大值域压缩为连续小索引

**复杂度**：
- 时间：O(n log n)，主要来自排序
- 空间：O(n)，存储映射表

**适用场景**：
- 值域大但实际数据点少（稀疏数据）
- 需要配合数组型数据结构使用
- 离线算法的预处理步骤

**实现关键**：
- 不要忘记去重
- 注意索引起始位置（0-indexed 或 1-indexed）
- 区间离散化时注意端点处理

坐标压缩虽然原理简单，但在算法竞赛和实际开发中应用极广，是每个程序员都应该掌握的基础技巧。
