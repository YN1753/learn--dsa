# 选择排序

## 概念解释

选择排序（Selection Sort）是一种简单直观的排序算法。它的核心思想是：**每一轮从未排序的部分中找到最小的元素，然后把它放到已排序部分的末尾**。

你可以把它想象成在一堆扑克牌中反复挑选最小的牌，然后按顺序摆好。

## 为什么重要

选择排序虽然效率不高，但它有重要的学习价值：

- **思想简单**：是最容易理解的排序算法之一
- **原地排序**：不需要额外的数组空间
- **交换次数少**：最多进行 n-1 次交换，适合写入代价高的场景
- **教学意义**：是学习更高级排序算法（堆排序）的基础

## 核心原理

算法步骤如下：

1. 在整个数组中找到最小元素，与第一个位置的元素交换
2. 在剩余的未排序部分中找到最小元素，与第二个位置的元素交换
3. 重复这个过程，直到整个数组有序

```
初始: [64, 25, 12, 22, 11]

第1轮: 找到最小值 11，与第1个元素交换
       [11, 25, 12, 22, 64]
       ↑ 已排序

第2轮: 在剩余部分找到最小值 12，与第2个元素交换
       [11, 12, 25, 22, 64]
       ↑↑ 已排序

第3轮: 找到最小值 22，交换
       [11, 12, 22, 25, 64]
       ↑↑↑ 已排序

第4轮: 找到最小值 25，已在正确位置
       [11, 12, 22, 25, 64]
       ↑↑↑↑ 已排序
```

## 复杂度分析

| 情况 | 时间复杂度 | 说明 |
|------|-----------|------|
| 最好情况 | O(n²) | 即使已排序，仍需扫描所有元素 |
| 最坏情况 | O(n²) | 完全逆序 |
| 平均情况 | O(n²) | 任何情况都是两层循环 |

- **空间复杂度**：O(1)，原地排序
- **是否稳定**：**不稳定**。交换操作可能改变相等元素的相对顺序

## 不稳定的原因

例如数组 `[5a, 8, 5b, 2, 9]`：

- 第1轮找到最小值 2，与 5a 交换 → `[2, 8, 5b, 5a, 9]`
- 此时 5a 和 5b 的相对顺序已经改变了（5b 在 5a 前面了）

这是选择排序不稳定的根本原因。

## 可视化说明

在可视化界面中：

- **蓝色柱子**：未排序的元素
- **绿色柱子**：已排序的元素
- **红色柱子**：当前正在比较的最小值候选
- **黄色柱子**：本轮扫描到的元素

通过动画可以直观看到每一轮如何"选出"最小值并放到正确位置。

## 常见错误

1. **混淆内外循环的范围**：外层循环只需要到 n-1（最后一个元素自然有序），内层循环从 i+1 开始
2. **忘记交换**：找到最小值后必须与当前位置交换
3. **误以为是稳定排序**：选择排序是不稳定的
4. **和冒泡排序混淆**：冒泡排序是相邻元素交换，选择排序是找到最小值后交换

## 实际应用

选择排序在以下场景中有用：

- **小规模数据**：元素数量很少时，简单实现比复杂算法更实用
- **交换代价高**：因为最多只做 n-1 次交换，适合交换操作代价很大的情况
- **教学演示**：帮助初学者理解排序的基本思想

在实际工程中，选择排序很少直接使用，通常会选择更高效的算法如快速排序、归并排序等。

## 总结

选择排序是一种简单但低效的排序算法，时间复杂度在任何情况下都是 O(n²)。它的优点是实现简单、交换次数少；缺点是效率低且不稳定。虽然实际应用不多，但它是理解排序算法思想的重要基础。
