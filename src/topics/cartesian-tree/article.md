# 笛卡尔树 (Cartesian Tree)

## 概念解释

笛卡尔树是一种**二叉树**，它同时满足两个关键性质：

1. **BST性质（二叉搜索树）**：如果对树进行中序遍历，得到的序列与原始序列顺序一致
2. **堆性质**：每个节点的值（priority）满足堆序关系——父节点的值小于等于（最小堆）或大于等于（最大堆）子节点的值

给定一个序列，笛卡尔树的结构是**唯一确定**的。这意味着同一个序列只能构建出一棵笛卡尔树。

### 直观理解

可以这样理解笛卡尔树的构建过程：

- 序列中的每个元素有两个属性：**位置（key）** 和 **值（priority）**
- key决定了元素在中序遍历中的位置（即原始顺序）
- priority决定了元素在树中的层次关系（堆序）

```
序列: [3, 1, 4, 1, 5, 9]
下标:  0  1  2  3  4  5

如果以值为priority（最小堆），以位置为key：
         1(1)
        /     \
      3(0)    1(3)
               \
               4(2)
              /   \
            5(4)  9(5)
```

## 为什么重要

笛卡尔树在算法竞赛和实际应用中有重要地位：

1. **RMQ问题的桥梁**：笛卡尔树将区间最值查询（RMQ）转化为最近公共祖先（LCA）问题，实现O(n)预处理、O(1)查询
2. **Treap的基础**：Treap（树堆）本质上就是随机化的笛卡尔树，是平衡树的经典实现
3. **O(n)构建**：使用单调栈可以在O(n)时间内构建笛卡尔树，非常高效
4. **独特的结构性质**：笛卡尔树的中序遍历就是原始序列，这一性质在很多场景下非常有用

## 核心原理

### 笛卡尔树的定义

给定序列 `a[0], a[1], ..., a[n-1]`，笛卡尔树是满足以下条件的二叉树：

1. 每个节点存储一个序列元素
2. 中序遍历的结果与原始序列相同（BST性质）
3. 父节点的值小于等于子节点的值（最小堆性质）

### O(n) 单调栈构建算法

笛卡尔树最经典的构建方法是使用**单调栈**，时间复杂度为O(n)。

**核心思想**：

按序列顺序从左到右插入元素，维护一个栈保存当前树的**右链**（从根到最右节点的路径）。栈中元素的priority保持递增（最小堆）。

**算法步骤**：

1. 初始化空栈
2. 对于每个新元素 `a[i]`：
   - 从栈顶开始，弹出所有 priority 大于 `a[i]` 的元素
   - 最后一个弹出的元素成为 `a[i]` 的左子节点
   - 如果栈不为空，栈顶元素的右子节点指向 `a[i]`
   - 将 `a[i]` 压入栈中
3. 最终栈底的元素就是根节点

```typescript
interface TreeNode {
  value: number   // priority
  index: number   // key
  left: TreeNode | null
  right: TreeNode | null
}

function buildCartesianTree(arr: number[]): TreeNode | null {
  const n = arr.length
  if (n === 0) return null

  const nodes: TreeNode[] = arr.map((val, i) => ({
    value: val,
    index: i,
    left: null,
    right: null,
  }))

  const stack: TreeNode[] = []

  for (let i = 0; i < n; i++) {
    let lastPopped: TreeNode | null = null

    // 弹出所有 priority 大于当前元素的栈顶元素
    while (stack.length > 0 && stack[stack.length - 1].value > arr[i]) {
      lastPopped = stack.pop()!
    }

    // 最后弹出的元素成为当前节点的左子树
    nodes[i].left = lastPopped

    // 如果栈不为空，栈顶元素的右子节点指向当前节点
    if (stack.length > 0) {
      stack[stack.length - 1].right = nodes[i]
    }

    // 将当前节点压入栈
    stack.push(nodes[i])
  }

  // 栈底元素即为根节点
  return stack[0]
}
```

### 为什么时间复杂度是O(n)？

关键观察：每个元素**最多入栈一次、出栈一次**。

- 入栈次数总计：n
- 出栈次数总计：最多n
- 因此总操作次数为O(n)

### 笛卡尔树与RMQ的关系

这是笛卡尔树最深刻的应用：

**定理**：对于数组 `a[0..n-1]`，如果以下标为key、以值为priority构建笛卡尔树，那么对于任意 `i, j`，`a[i..j]` 的最值对应的元素就是节点 `i` 和节点 `j` 的**最近公共祖先（LCA）**对应的元素。

**证明思路**：
- 设 `k = LCA(i, j)`，则 `i < k < j`（因为中序遍历有序）
- 由于 `k` 是 `i` 和 `j` 的祖先，根据堆性质，`a[k]` 是 `a[i..j]` 中的最值

这使得RMQ问题可以转化为LCA问题：

```
原数组: [3, 1, 4, 1, 5, 9]
RMQ(2, 5) = min(a[2..5]) = min(4, 1, 5, 9) = 1
LCA(2, 5) 在笛卡尔树中对应的值就是 1
```

### 时间复杂度总结

| 操作 | 时间复杂度 | 说明 |
|------|------------|------|
| 构建笛卡尔树 | O(n) | 单调栈方法 |
| RMQ查询（预处理） | O(n) | 构建笛卡尔树 |
| RMQ查询（单次） | O(1) | 通过LCA（ST表） |
| 空间复杂度 | O(n) | 存储树节点 |

## 常见错误

### 1. 混淆key和priority

```typescript
// 错误：将值同时作为key和priority
// 正确：key是下标（决定中序遍历），priority是值（决定堆序）
```

笛卡尔树有两个维度：key决定位置，priority决定层次。在RMQ应用中，下标是key，数组值是priority。

### 2. 忘记处理左子树

```typescript
// 错误：直接将新元素压入栈，不处理弹出的元素
while (stack.length > 0 && stack[stack.length - 1].value > arr[i]) {
  stack.pop() // 弹出但没有保存
}

// 正确：保存最后弹出的元素，作为新元素的左子树
let lastPopped: TreeNode | null = null
while (stack.length > 0 && stack[stack.length - 1].value > arr[i]) {
  lastPopped = stack.pop()!
}
nodes[i].left = lastPopped
```

### 3. 混淆最小堆和最大堆

笛卡尔树可以是最小堆或最大堆，取决于问题需求：

- **最小堆**：父节点值 <= 子节点值，用于区间最小值查询
- **最大堆**：父节点值 >= 子节点值，用于区间最大值查询

构建时需要相应调整单调栈的方向。

### 4. 认为笛卡尔树是平衡的

笛卡尔树**不保证平衡**。对于有序序列，笛卡尔树会退化为链。例如序列 `[1, 2, 3, 4, 5]` 构建的笛卡尔树是一条右链。

但在Treap中，通过随机化priority，可以获得期望O(log n)的高度。

## 实际应用

### 1. 区间最值查询 (RMQ)

笛卡尔树最重要的应用。通过将RMQ转化为LCA，实现：
- 预处理：O(n)构建笛卡尔树 + O(n log n)构建LCA数据结构
- 查询：O(1)

这是理论上最优的RMQ解法之一。

### 2. Treap（树堆）

Treap是笛卡尔树的随机化版本：
- key由用户指定（如插入的值）
- priority随机生成
- 期望高度O(log n)

Treap支持高效的查找、插入、删除操作，实现简单，在算法竞赛中广泛使用。

### 3. 可持久化数据结构

笛卡尔树的结构可以用于构建可持久化数据结构，支持历史版本查询。

### 4. 序列分析

笛卡尔树可以用于分析序列的结构特征，如：
- 识别序列的"峰"和"谷"
- 分析序列的递增/递减趋势
- 在生物信息学中用于序列比对

### 5. 直方图最大矩形问题

笛卡尔树可以用于解决直方图中最大矩形问题，这是RMQ应用的一个具体实例。

## 总结

笛卡尔树是一种兼具理论深度和实用价值的数据结构：

**核心性质**：
- 同时满足BST性质和堆性质
- 给定序列的笛卡尔树结构唯一
- 可以在O(n)时间内用单调栈构建

**主要应用**：
- RMQ问题的最优解法（O(n)预处理，O(1)查询）
- Treap（随机化平衡树）的理论基础
- 序列结构分析

**与Treap的关系**：
- Treap = 随机化的笛卡尔树
- Treap用随机priority保证期望平衡
- 笛卡尔树用确定性priority可能退化

**适用场景**：
- 需要高效区间最值查询的场景
- 需要简单实现的平衡树（Treap）
- 需要同时维护序列顺序和优先级的场景

理解笛卡尔树有助于深入理解二叉搜索树、堆、RMQ等多个数据结构之间的内在联系。
