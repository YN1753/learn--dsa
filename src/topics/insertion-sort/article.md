# 插入排序 (Insertion Sort)

## 概念解释

插入排序是一种简单直观的排序算法，它的工作方式类似于我们整理扑克牌：每次从手中取一张牌，将其插入到已排好序的牌的正确位置。

**核心思想**：将数组分为"已排序"和"未排序"两部分，每次从未排序部分取出第一个元素，插入到已排序部分的正确位置。

### 核心术语

- **已排序部分（Sorted Portion）**：数组左侧已经排好序的子数组，初始时只包含第一个元素
- **未排序部分（Unsorted Portion）**：数组右侧尚未处理的元素
- **插入点（Insertion Point）**：当前元素在已排序部分中应该放置的位置
- **稳定排序（Stable Sort）**：相等元素的相对顺序在排序前后保持不变

### 算法步骤

1. 从第二个元素开始（索引 1），将当前元素标记为"待插入元素"
2. 将待插入元素与已排序部分的元素从右到左逐一比较
3. 如果已排序元素大于待插入元素，将该元素右移一位
4. 找到正确位置后，将待插入元素放入该位置
5. 重复以上步骤，直到所有元素都被处理

```
初始: [5, 3, 8, 1, 2]

第1步: [3, 5, 8, 1, 2]  -- 将3插入到5前面
第2步: [3, 5, 8, 1, 2]  -- 8已在正确位置
第3步: [1, 3, 5, 8, 2]  -- 将1插入到最前面
第4步: [1, 2, 3, 5, 8]  -- 将2插入到1和3之间
```

## 为什么重要

### 近乎有序数据的最优选择

插入排序在近乎有序的数组上表现极佳，时间复杂度接近 O(n)。这是因为内层循环几乎不需要移动元素，只需少量比较即可找到插入点。

### 小规模数据的首选

对于小规模数据（n < 50），插入排序的常数因子非常小，实际运行速度往往超过 O(n log n) 的高级排序算法。许多高级排序算法（如 Timsort、Introsort）在小规模子数组上会切换到插入排序。

### 在线算法

插入排序是一种**在线算法**（Online Algorithm）：可以在数据到达时逐步处理，无需等待所有数据到齐。这在流式数据处理中非常有用。

### 稳定排序

插入排序是稳定的，相等元素的相对顺序不会改变。这在多字段排序中很重要。

### 基础算法教学

插入排序是理解排序算法思想的最佳起点之一，它的"增量插入"思想是许多高级算法的基础。

## 核心原理

### 基本实现

```typescript
function insertionSort(arr: number[]): number[] {
  const result = [...arr]
  for (let i = 1; i < result.length; i++) {
    const key = result[i]
    let j = i - 1
    while (j >= 0 && result[j] > key) {
      result[j + 1] = result[j]
      j--
    }
    result[j + 1] = key
  }
  return result
}
```

### 详细过程分析

以数组 `[5, 3, 8, 1, 2]` 为例：

**第 1 轮（i=1）**：待插入元素 key=3
- 比较 5 > 3，5 右移 → `[_, 5, 8, 1, 2]`
- 到达数组开头，插入 3 → `[3, 5, 8, 1, 2]`

**第 2 轮（i=2）**：待插入元素 key=8
- 比较 5 > 8？否，停止
- 8 已在正确位置 → `[3, 5, 8, 1, 2]`

**第 3 轮（i=3）**：待插入元素 key=1
- 比较 8 > 1，8 右移 → `[3, 5, _, 8, 2]`
- 比较 5 > 1，5 右移 → `[3, _, 5, 8, 2]`
- 比较 3 > 1，3 右移 → `[_, 3, 5, 8, 2]`
- 到达数组开头，插入 1 → `[1, 3, 5, 8, 2]`

**第 4 轮（i=4）**：待插入元素 key=2
- 比较 8 > 2，8 右移 → `[1, 3, 5, _, 8]`
- 比较 5 > 2，5 右移 → `[1, 3, _, 5, 8]`
- 比较 3 > 2，3 右移 → `[1, _, 3, 5, 8]`
- 比较 1 > 2？否，停止
- 插入 2 → `[1, 2, 3, 5, 8]`

### 时间复杂度分析

| 情况 | 时间复杂度 | 说明 |
|------|-----------|------|
| 最好情况 | O(n) | 数组已经有序，每次只需一次比较 |
| 平均情况 | O(n²) | 随机数据，平均需要 n²/4 次比较和移动 |
| 最坏情况 | O(n²) | 数组逆序，每次需要比较和移动所有已排序元素 |

### 空间复杂度

O(1) —— 插入排序是原地排序算法，只需要常数级别的额外空间。

### 与其它简单排序的比较

| 特性 | 插入排序 | 冒泡排序 | 选择排序 |
|------|---------|---------|---------|
| 最好时间 | O(n) | O(n) | O(n²) |
| 平均时间 | O(n²) | O(n²) | O(n²) |
| 最坏时间 | O(n²) | O(n²) | O(n²) |
| 稳定性 | 稳定 | 稳定 | 不稳定 |
| 交换次数 | O(n²) | O(n²) | O(n) |
| 适应性 | 有 | 有 | 无 |

**适应性（Adaptive）**：指算法能利用数据中已有的有序性。插入排序在近乎有序的数据上表现极佳，而选择排序无论数据如何都是 O(n²)。

## 常见错误

### 1. 内层循环条件错误

```typescript
// 错误：忘记检查 j >= 0
while (arr[j] > key) {  // j 可能变为 -1，导致越界！
  arr[j + 1] = arr[j]
  j--
}

// 正确：同时检查边界和比较条件
while (j >= 0 && arr[j] > key) {
  arr[j + 1] = arr[j]
  j--
}
```

### 2. 插入位置计算错误

```typescript
// 错误：使用 j 而不是 j + 1
arr[j] = key  // 错误！j 停留在最后一个需要右移的元素上

// 正确：插入位置是 j + 1
arr[j + 1] = key  // j + 1 是正确的插入位置
```

### 3. 从索引 0 开始遍历

```typescript
// 错误：从索引 0 开始
for (let i = 0; i < arr.length; i++) {  // 第一个元素不需要排序

// 正确：从索引 1 开始
for (let i = 1; i < arr.length; i++) {  // 第一个元素默认已排序
```

### 4. 误以为插入排序效率很低

插入排序在以下场景中表现优异：
- 小规模数据（n < 50）
- 近乎有序的数据
- 流式数据（在线算法）

不要在所有场景下都使用 O(n log n) 的排序算法，插入排序在特定场景下可能是更好的选择。

## 实际应用

### 1. Timsort 的组成部分

Python 和 Java 的标准排序算法 Timsort 在小规模子数组（通常 ≤ 32 或 64 个元素）上使用插入排序。这是因为插入排序在小规模数据上的常数因子很小，比归并排序的递归开销更高效。

### 2. Introsort 的回退策略

Introsort（C++ STL 的 sort 实现）在递归深度过大时切换到堆排序，在小规模子数组上切换到插入排序。

### 3. 在线数据处理

当数据逐个到达时（如实时传感器数据），插入排序可以逐步维护一个有序序列，无需等待所有数据到齐。

### 4. 链表排序

插入排序特别适合链表排序，因为链表的插入操作只需要修改指针，不需要移动元素。而快速排序和堆排序在链表上效率较低。

### 5. 几乎有序的数据

在日志文件按时间排序但有少量乱序的情况下，插入排序的效率接近 O(n)，远优于其他排序算法。

## 总结

插入排序是一种简单但强大的排序算法，核心思想是"增量插入"：

- **时间复杂度**：平均 O(n²)，最好 O(n)（近乎有序数据）
- **空间复杂度**：O(1)，原地排序
- **稳定性**：稳定，相等元素的相对顺序不变
- **适应性**：能利用数据中已有的有序性
- **适用场景**：小规模数据、近乎有序数据、链表排序、在线算法

虽然插入排序在大规模随机数据上不如 O(n log n) 算法，但它在特定场景下的优秀表现使其成为算法工具箱中不可或缺的工具。理解插入排序，也是理解 Timsort、Introsort 等工业级排序算法的基础。
