# 埃氏筛 (Sieve of Eratosthenes)

## 概念解释

埃氏筛法（Sieve of Eratosthenes）是一种高效找出所有小于等于 n 的质数的经典算法，由古希腊数学家埃拉托斯特尼（Eratosthenes）在公元前 276 年左右提出。

**核心思想**：从 2 开始，将每个质数的倍数标记为合数，剩下的未标记数就是质数。

### 基本术语

| 术语 | 说明 |
|------|------|
| 质数 (Prime) | 大于 1 且只能被 1 和自身整除的自然数 |
| 合数 (Composite) | 大于 1 且不是质数的自然数 |
| 筛选 (Sieve) | 逐个排除合数的过程 |
| 最小质因子 (LPF) | 一个合数的最小质因数 |

### 算法直觉

想象你有一张写满数字的表格：

1. 从 2 开始，它是质数
2. 把 2 的所有倍数（4, 6, 8, 10...）划掉
3. 找到下一个没被划掉的数 3，它是质数
4. 把 3 的所有倍数（9, 12, 15...）划掉
5. 重复这个过程，直到处理完 sqrt(n)

## 为什么重要

埃氏筛在算法竞赛和实际工程中都非常重要：

1. **高效求质数**：比逐个判断快得多，适合批量处理
2. **质因数分解基础**：预处理后可以在 O(log n) 时间分解任意数
3. **密码学基础**：RSA 等加密算法依赖大质数的生成和判断
4. **数论问题核心工具**：欧拉函数、莫比乌斯函数等都依赖质数筛选

## 核心原理

### 埃氏筛算法

```typescript
function sieveOfEratosthenes(n: number): number[] {
  // 初始化：假设所有数都是质数
  const isPrime = new Array(n + 1).fill(true)
  isPrime[0] = isPrime[1] = false

  // 从 2 开始筛选
  for (let i = 2; i * i <= n; i++) {
    if (isPrime[i]) {
      // 从 i² 开始标记合数
      for (let j = i * i; j <= n; j += i) {
        isPrime[j] = false
      }
    }
  }

  // 收集所有质数
  const primes: number[] = []
  for (let i = 2; i <= n; i++) {
    if (isPrime[i]) primes.push(i)
  }
  return primes
}
```

**关键优化**：
- 外层循环只需到 sqrt(n)，因为大于 sqrt(n) 的质数的倍数在之前已经被筛掉了
- 内层循环从 i² 开始，因为小于 i² 的合数已被更小的质数筛掉

**时间复杂度**：O(n log log n)

由梅滕斯定理（Mertens' theorem）可知，调和级数 1/2 + 1/3 + 1/5 + 1/7 + ... 约等于 log log n，因此总操作次数约为 n × log log n。

### 线性筛（欧拉筛）

埃氏筛的缺点是同一个合数可能被多次标记（例如 12 被 2 和 3 各标记一次）。线性筛通过保证每个合数只被其最小质因子筛掉一次，实现了 O(n) 的时间复杂度。

```typescript
function linearSieve(n: number): number[] {
  const primes: number[] = []
  const lpf = new Array(n + 1).fill(0) // 最小质因子

  for (let i = 2; i <= n; i++) {
    if (lpf[i] === 0) {
      // i 是质数
      lpf[i] = i
      primes.push(i)
    }
    // 用已知质数筛掉合数
    for (const p of primes) {
      if (p > lpf[i] || i * p > n) break
      lpf[i * p] = p // 标记 i*p 的最小质因子为 p
    }
  }
  return primes
}
```

**核心思想**：每个合数 n 只被它的最小质因子筛掉。通过 `p > lpf[i]` 的判断保证了这一点。

### 质因数分解

利用线性筛预处理的最小质因子数组（LPF），可以高效地进行质因数分解：

```typescript
function factorize(n: number, lpf: number[]): Map<number, number> {
  const factors = new Map<number, number>()
  while (n > 1) {
    const p = lpf[n]
    factors.set(p, (factors.get(p) || 0) + 1)
    n /= p
  }
  return factors
}

// 例：factorize(12, lpf) => {2: 2, 3: 1}，即 12 = 2² × 3
```

**时间复杂度**：O(log n)，因为每次至少将 n 除以 2。

### 质数定理

质数定理（Prime Number Theorem）描述了质数在自然数中的分布规律：

> 小于等于 n 的质数个数 π(n) ≈ n / ln(n)

当 n 越大时，近似越准确：

| n | π(n) 实际值 | n/ln(n) 近似值 | 比值 |
|---|------------|---------------|------|
| 100 | 25 | 22 | 1.14 |
| 1,000 | 168 | 145 | 1.16 |
| 10,000 | 1,229 | 1,086 | 1.13 |
| 100,000 | 9,592 | 8,686 | 1.10 |

通过动画可以直观地看到：

- 筛选效率随着质数增大而提高（大质数的倍数更少）

## 常见错误

### 1. 忘记从 i² 开始

```typescript
// 错误：从 2*i 开始，导致重复标记
for (let j = 2 * i; j <= n; j += i) { ... }

// 正确：从 i² 开始，避免重复
for (let j = i * i; j <= n; j += i) { ... }
```

### 2. 外层循环范围错误

```typescript
// 错误：循环到 n，浪费时间
for (let i = 2; i <= n; i++) { ... }

// 正确：只需到 sqrt(n)
for (let i = 2; i * i <= n; i++) { ... }
```

### 3. 线性筛中判断条件错误

```typescript
// 错误：忘记 p > lpf[i] 的判断
for (const p of primes) {
  lpf[i * p] = p  // 可能用非最小质因子标记
}

// 正确：保证用最小质因子标记
for (const p of primes) {
  if (p > lpf[i] || i * p > n) break
  lpf[i * p] = p
}
```

### 4. 整数溢出

```typescript
// 错误：i * i 可能溢出（C/C++ 中常见）
for (int i = 2; i * i <= n; i++) { ... }

// 正确：使用 long long 或改写条件
for (long long i = 2; i * i <= n; i++) { ... }
```

## 实际应用

### 1. 密码学中的质数生成

RSA 加密算法需要生成大质数。虽然埃氏筛不能直接生成非常大的质数，但它可以预筛选掉小质数的倍数，加速后续的素性测试。

### 2. 区间筛法

当需要求 [L, R] 区间内的质数且 R 很大但 R-L 不大时，可以先用埃氏筛求出 sqrt(R) 以内的质数，再用这些质数去筛 [L, R] 区间。

### 3. 欧拉函数打表

利用筛法可以 O(n) 预处理出所有数的欧拉函数值，在数论问题中非常有用。

### 4. 竞赛中的质数表

在算法竞赛中，预先用筛法生成质数表是常见操作，可以在 O(1) 时间判断一个数是否为质数。

## 总结

埃氏筛是数论中最基础、最重要的算法之一：

**核心要点**：
- **基本思想**：从小到大，用质数的倍数筛掉合数
- **时间复杂度**：O(n log log n)，非常高效
- **优化关键**：外层到 sqrt(n)，内层从 i² 开始
- **线性筛**：O(n)，每个合数只被最小质因子筛一次

**算法选择**：
- 数据量不大（n < 10^7）：普通埃氏筛足够
- 需要最小质因子：使用线性筛
- 超大范围但区间小：使用区间筛法

**适用场景**：
- 批量求质数
- 质因数分解
- 欧拉函数打表
- 密码学中的质数预筛选

掌握筛法是深入学习数论的第一步，也是算法竞赛中的必备技能。
