# AC自动机 (Aho-Corasick)

## 概念解释

AC自动机（Aho-Corasick Automaton）是一种经典的多模式串匹配算法，由 Alfred V. Aho 和 Margaret J. Corasick 于 1975 年提出。它能够在**一次扫描**文本的过程中，同时匹配多个模式串，是单模式匹配算法 KMP 的自然推广。

**核心思想：** AC自动机将所有模式串构建成一棵 Trie 树，然后在 Trie 上计算**失败指针**（failure link），使得匹配失败时能够快速跳转到下一个可能匹配的位置，而不需要回退文本指针。

```
模式串集合: {"he", "she", "his", "hers"}

构建的 Trie:
        root
       / | \
      h  s  
     / \  \
    i   e  h
   (E) (E)  \
             e (E)

AC自动机在 Trie 基础上增加了失败指针（虚线箭头），
使得匹配失败时能快速跳转到最长后缀匹配位置。
```

### 关键特性

- **基于 Trie**：所有模式串存储在 Trie 中，共享公共前缀
- **失败指针**：每个节点有一条失败指针，指向 Trie 中该节点对应字符串的最长后缀所对应的节点
- **输出链接**：某些节点通过输出链接指向其他模式串的结尾节点（当一个模式串是另一个模式串的后缀时）
- **一次扫描**：遍历文本一次即可找到所有匹配，不需要回退文本指针

## 为什么重要

AC自动机在实际工程中有广泛应用：

- **敏感词过滤**：社交平台、论坛、直播弹幕等需要实时检测大量敏感词，AC自动机可以在一次扫描中完成所有敏感词的匹配
- **杀毒引擎**：病毒特征码匹配需要同时检测数千种已知病毒签名，AC自动机提供了高效的解决方案
- **入侵检测系统（IDS）**：网络流量分析需要匹配大量攻击特征规则，Snort 等系统使用了 AC自动机的变体
- **搜索引擎**：在索引阶段需要对大量关键词进行匹配和提取
- **生物信息学**：DNA/RNA 序列中同时搜索多个基因片段

相比逐个使用 KMP 匹配每个模式串（O(n * m)），AC自动机只需 O(n + m + z) 的时间，其中 z 是匹配次数。

## 核心原理

### 第一步：构建 Trie

将所有模式串插入 Trie 树。每个节点代表一个字符，从根到某节点的路径构成一个前缀。

```
插入 "he", "she", "his", "hers":

        root
       / | \
      h  s  
     / \  \
    i   e  h
   [E] [E]  \
             e [E]
              |
              r
              |
              s [E]

[E] 表示该节点是一个模式串的结尾
```

时间复杂度：O(m)，其中 m 是所有模式串的总长度。

### 第二步：BFS 计算失败指针

使用 BFS（广度优先搜索）从根节点开始逐层计算每个节点的失败指针。

**失败指针的含义：** 节点 v 的失败指针指向 Trie 中，代表 v 对应字符串的**最长真后缀**的节点。

**计算规则：**

1. 根节点的所有子节点的失败指针指向根
2. 对于其他节点 v（父节点为 u，经过字符 c 到达 v）：
   - 令 f = u 的失败指针指向的节点
   - 如果 f 有经过字符 c 的子节点 w，则 v 的失败指针指向 w
   - 否则，继续沿 f 的失败指针向上跳转，直到找到有 c 子节点的节点或到达根

```
BFS 计算失败指针的过程：

第1层: h→root, s→root
第2层: i→(h的失败指针root有子节点h? 没有i, 所以→root)
       e→(h的失败指针root有子节点s? 没有e, →root)
       h→(s的失败指针root有子节点h? 有! →h节点)
第3层: 依次类推...
```

时间复杂度：O(m)，其中 m 是 Trie 中所有节点的数量。

### 第三步：匹配过程

从文本的第一个字符开始，沿着 Trie 边匹配：

1. 如果当前节点有对应字符的子节点，沿子节点前进
2. 如果没有，沿失败指针跳转，直到找到有该字符子节点的节点或回到根
3. 每到达一个节点，检查该节点及通过输出链接可达的节点是否为模式串结尾

```
文本: "ushers"
模式串: {"he", "she", "his", "hers"}

位置 0: 'u' - 无匹配，停在 root
位置 1: 's' - 到达 s 节点
位置 2: 'h' - 到达 sh 节点
位置 3: 'e' - 到达 she 节点 → 匹配 "she"！
         沿输出链接 → 匹配 "he"！
位置 4: 'r' - 失败跳转到 root，再到 h→e，然后无 r 子节点，
         失败跳转到 root，无匹配
位置 5: 's' - 到达 s 节点，无匹配

总匹配: "she" 在位置 1-3, "he" 在位置 2-3
```

### 时间复杂度分析

- **建 Trie**：O(m)，m 为所有模式串总长度
- **计算失败指针**：O(m)
- **匹配**：O(n + z)，n 为文本长度，z 为匹配次数
- **总计**：O(n + m + z)

空间复杂度：O(m * |Σ|)，其中 |Σ| 是字符集大小。使用哈希表可优化为 O(m)。

## 常见错误

### 1. 失败指针构建顺序错误

**错误：** 使用 DFS 而非 BFS 来计算失败指针。

**正确做法：** 必须使用 BFS，因为计算某节点的失败指针时，其父节点的失败指针必须已经计算完毕。BFS 保证了层序遍历的顺序。

```
错误: DFS 遍历 → 可能在父节点失败指针未计算时就处理子节点
正确: BFS 遍历 → 保证从上到下逐层计算
```

### 2. 未正确传播输出链接

**错误：** 只检查当前节点是否为模式串结尾，忽略了通过失败指针链可以到达的其他模式串结尾。

**正确做法：** 当到达某节点时，需要沿失败指针链检查所有可达的模式串结尾。例如，当匹配到 "she" 时，也需要报告 "he"（因为 "he" 是 "she" 的后缀）。

### 3. 混淆 AC自动机与 KMP

**常见误区：** 认为 AC自动机和 KMP 是同一类算法。

**区别：**
- KMP 是**单模式**匹配：一次匹配一个模式串
- AC自动机是**多模式**匹配：一次匹配所有模式串
- AC自动机可以看作 KMP 在多模式场景下的推广
- KMP 的 next 数组对应 AC自动机中只有一条分支的特殊情况

### 4. 失败指针未指向根

**错误：** 在某些边界情况下，失败指针没有正确指向根节点。

**正确做法：** 根节点的所有直接子节点的失败指针必须指向根。当沿失败指针链回溯到根仍未找到匹配时，应停在根节点。

### 5. 字符集处理不当

**错误：** 假设字符集仅为小写英文字母，用固定大小数组实现。

**正确做法：** 实际应用中字符集可能很大（Unicode），应使用哈希表存储子节点，或者根据实际场景选择合适的实现方式。

## 实际应用

### 1. 敏感词过滤系统

```
场景: 社交平台评论过滤
模式串: 数千个敏感词
文本: 用户发布的评论

实现:
1. 系统启动时用所有敏感词构建 AC自动机
2. 每条评论只需扫描一次即可检测所有敏感词
3. 匹配到的敏感词用 * 号替换或标记审核

优势: 比逐个关键词搜索快数个数量级
```

### 2. 杀毒软件特征码匹配

```
场景: 病毒特征码扫描
模式串: 数万个病毒签名片段
文本: 文件的二进制内容

实现:
1. 将所有已知病毒特征码构建 AC自动机
2. 扫描文件内容，一次遍历匹配所有特征码
3. 命中的特征码标记为可疑文件
```

### 3. 网络入侵检测

```
场景: IDS/IPS 系统
模式串: 攻击特征规则（如 SQL 注入、XSS 模式）
文本: 网络数据包内容

实现:
1. 将所有攻击特征转换为字符串模式
2. 构建 AC自动机
3. 实时扫描网络流量，检测潜在攻击
```

### 4. 搜索引擎关键词提取

```
场景: 文档索引
模式串: 需要索引的关键词库
文本: 网页或文档内容

实现:
1. 用关键词库构建 AC自动机
2. 扫描文档内容，记录所有关键词出现的位置
3. 建立倒排索引
```

### 5. 生物信息学

```
场景: DNA 序列分析
模式串: 已知基因片段或标记序列
文本: 测序得到的 DNA 长序列

实现:
1. 将目标基因片段构建 AC自动机
2. 在长序列中一次扫描定位所有目标片段
3. 用于基因定位、变异检测等
```

## 总结

AC自动机是多模式串匹配的经典算法，它巧妙地将 Trie 树与失败指针结合，实现了在一次文本扫描中匹配所有模式串的高效算法。

**核心要点：**
- **Trie + 失败指针**：Trie 提供前缀匹配能力，失败指针在失配时提供快速跳转
- **BFS 构建**：失败指针必须通过 BFS 按层序计算
- **输出链接传播**：匹配时需沿失败指针链检查所有可达的模式串结尾
- **线性时间**：O(n + m + z) 的时间复杂度，其中 n 为文本长度，m 为模式串总长度，z 为匹配次数

**与其他算法的关系：**
- AC自动机是 KMP 的多模式推广
- AC自动机可以看作在 Trie 上运行的 KMP
- 当只有一个模式串时，AC自动机退化为 KMP

掌握 AC自动机，你就掌握了处理大规模多模式匹配问题的核心工具，在敏感词过滤、安全检测、搜索引擎等领域都有重要应用。
