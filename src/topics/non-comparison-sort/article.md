# 非比较排序 (Non-Comparison Sort)

## 概念解释

在学习排序算法时，我们通常接触的冒泡排序、快速排序、归并排序等都是**基于比较的排序**——它们通过元素之间的两两比较来确定顺序。计算机科学中有一个重要结论：任何基于比较的排序算法，时间复杂度下界为 **O(n log n)**。

### 为什么有 O(n log n) 下界？

基于比较的排序可以用一棵**决策树**来表示。n 个元素有 n! 种排列，排序算法必须能区分所有这些情况。一棵二叉决策树的叶子数至少为 n!，因此树的高度（即最坏比较次数）至少为 log₂(n!) ≈ n log n。

### 非比较排序的核心思想

非比较排序绕过了这个限制——它们**不通过比较来确定顺序**，而是利用元素本身的值（如数值大小、数位等）直接计算元素应该放置的位置。

- **计数排序**：将元素的值作为计数数组的索引，统计每个值出现的次数
- **基数排序**：逐位处理数字，从最低位到最高位依次排序
- **桶排序**：将元素分配到不同的桶中，每个桶内单独排序

```
比较排序：  "A 和 B 谁大？" → 交换/移动
非比较排序："A 的值是 5，放到位置 5" → 直接定位
```

## 为什么重要

### 突破 O(n log n) 瓶颈

当数据的值域有限时（如排序 0-100 之间的 100 万个整数），计数排序可以在 O(n) 时间内完成，远快于快速排序的 O(n log n)。在大规模数据场景下，这个差距非常显著。

### 稳定排序

计数排序和基数排序都是**稳定排序**——相等元素的相对顺序在排序后保持不变。这在多关键字排序中非常重要：例如先按成绩排序，再按姓名排序，稳定排序能保证成绩相同时按姓名排列。

### 基数排序的基石

基数排序依赖于一个稳定的子排序算法（通常是计数排序）。理解计数排序是掌握基数排序的前提。

### 实际工程中的应用

数据库的字符串排序、IP 地址排序、大整数排序等场景中，非比较排序经常被使用或作为底层优化。

## 核心原理

### 计数排序 (Counting Sort)

计数排序是最简单的非比较排序。核心思想是：统计每个值出现的次数，然后依次输出。

**算法步骤**：

1. 找到数组中的最大值 max 和最小值 min，确定范围 k = max - min + 1
2. 创建一个大小为 k 的计数数组 count[]，初始化为 0
3. 遍历原数组，对每个元素 x，执行 count[x - min]++
4. 对计数数组计算前缀和（累积计数）
5. 从后往前遍历原数组，根据前缀和确定每个元素的输出位置，放入结果数组

```
原数组: [4, 2, 2, 8, 3, 3, 1]

第1步 统计次数:
  值:    1  2  3  4  8
  次数:  1  2  2  1  1

第2步 前缀和:
  累积:  1  3  5  6  7
  (含义: 值 ≤ 1 的有1个, ≤ 2 的有3个, ...)

第3步 从后往前放置:
  元素 1 → 位置 count[0]-1 = 0  → 输出[0]=1
  元素 3 → 位置 count[2]-1 = 4  → 输出[4]=3
  元素 3 → 位置 count[2]-1 = 3  → 输出[3]=3
  ...（依次放置，保持稳定性）

结果: [1, 2, 2, 3, 3, 4, 8]
```

**关键细节**：从后往前遍历原数组是为了保证稳定性。如果从前往后遍历，相同值的元素会逆序放置。

### 基数排序 (Radix Sort)

基数排序处理位数较多的整数。它不直接比较数字大小，而是按每一位进行排序。

**算法步骤**：

1. 找到最大值，确定最大位数 d
2. 从最低位（个位）开始，到最高位，逐位进行稳定排序
3. 每一轮排序后，数组按当前位有序，且保持低位的相对顺序

```
原数组: [170, 45, 75, 90, 802, 24, 2, 66]

第1轮 按个位排序:
  170, 90 → 个位 0
  802, 2  → 个位 2
  24      → 个位 4
  45, 75  → 个位 5
  66      → 个位 6
  结果: [170, 90, 802, 2, 24, 45, 75, 66]

第2轮 按十位排序:
  802, 2  → 十位 0
  24      → 十位 2
  45      → 十位 4
  66      → 十位 6
  170, 75 → 十位 7
  90      → 十位 9
  结果: [802, 2, 24, 45, 66, 170, 75, 90]

第3轮 按百位排序:
  2, 24, 45, 66, 75, 90 → 百位 0
  170 → 百位 1
  802 → 百位 8
  结果: [2, 24, 45, 66, 75, 90, 170, 802]
```

**LSD vs MSD**：
- **LSD（最低位优先）**：从个位开始向高位排序。必须使用稳定排序。最常用。
- **MSD（最高位优先）**：从最高位开始向低位排序。类似字典序，可以提前终止，但实现更复杂。

### 桶排序 (Bucket Sort)

桶排序将数据分配到有限数量的"桶"中，每个桶内单独排序，最后合并。

**算法步骤**：

1. 确定桶的数量和每个桶的范围
2. 将每个元素分配到对应的桶中
3. 对每个桶内的元素单独排序（可以使用插入排序等）
4. 按桶的顺序合并所有桶中的元素

```
原数组: [0.42, 0.32, 0.23, 0.52, 0.25, 0.47, 0.51]

桶数量 = 5, 每桶范围 0.2

桶[0.0-0.2]:  (空)
桶[0.2-0.4]:  [0.32, 0.23, 0.25]
桶[0.4-0.6]:  [0.42, 0.52, 0.47, 0.51]

桶内排序:
  桶[0.2-0.4]:  [0.23, 0.25, 0.32]
  桶[0.4-0.6]:  [0.42, 0.47, 0.51, 0.52]

合并: [0.23, 0.25, 0.32, 0.42, 0.47, 0.51, 0.52]
```

### 复杂度对比

| 算法 | 时间(平均) | 时间(最坏) | 空间 | 稳定 | 适用场景 |
|------|-----------|-----------|------|------|---------|
| 计数排序 | O(n+k) | O(n+k) | O(n+k) | 是 | 值域小的整数 |
| 基数排序 | O(d·(n+k)) | O(d·(n+k)) | O(n+k) | 是 | 位数有限的整数 |
| 桶排序 | O(n) | O(n²) | O(n) | 视子排序 | 均匀分布的数据 |

其中 n 是元素数量，k 是值域范围，d 是最大位数。

## 常见错误

### 1. 计数排序不使用前缀和导致不稳定

```typescript
// ❌ 错误：直接根据计数放置，不稳定
function countingSortUnstable(arr: number[]): number[] {
  const max = Math.max(...arr)
  const count = new Array(max + 1).fill(0)
  for (const x of arr) count[x]++
  const result: number[] = []
  for (let i = 0; i <= max; i++) {
    while (count[i]-- > 0) result.push(i)
  }
  return result
  // 相同值的元素顺序被破坏！
}

// ✅ 正确：使用前缀和，从后往前放置
function countingSortStable(arr: number[]): number[] {
  const max = Math.max(...arr)
  const min = Math.min(...arr)
  const k = max - min + 1
  const count = new Array(k).fill(0)
  for (const x of arr) count[x - min]++
  // 前缀和
  for (let i = 1; i < k; i++) count[i] += count[i - 1]
  const result = new Array(arr.length)
  // 从后往前，保证稳定性
  for (let i = arr.length - 1; i >= 0; i--) {
    result[--count[arr[i] - min]] = arr[i]
  }
  return result
}
```

### 2. 基数排序使用不稳定的子排序

```typescript
// ❌ 错误：使用选择排序作为子排序（不稳定）
// 按个位排序 [32, 21, 13] → [21, 32, 13]（正确）
// 按十位排序时选择排序可能打乱个位顺序 → 结果错误

// ✅ 正确：必须使用稳定排序，如计数排序
function countingSortByDigit(arr: number[], exp: number): number[] {
  const count = new Array(10).fill(0)
  for (const x of arr) count[Math.floor(x / exp) % 10]++
  for (let i = 1; i < 10; i++) count[i] += count[i - 1]
  const result = new Array(arr.length)
  for (let i = arr.length - 1; i >= 0; i--) {
    const digit = Math.floor(arr[i] / exp) % 10
    result[--count[digit]] = arr[i]
  }
  return result
}
```

### 3. 值域过大导致计数排序空间爆炸

```typescript
// ❌ 错误：值域 [0, 10^9] 时使用计数排序
const arr = [1000000000, 999999999, 1]
// count 数组需要 10^9 个元素！内存溢出！

// ✅ 正确：值域大时应使用基数排序或比较排序
function radixSort(arr: number[]): number[] {
  const max = Math.max(...arr)
  for (let exp = 1; Math.floor(max / exp) > 0; exp *= 10) {
    arr = countingSortByDigit(arr, exp)
  }
  return arr
}
```

### 4. 忘记处理负数

```typescript
// ❌ 错误：计数排序假设所有值 ≥ 0
const count = new Array(max + 1).fill(0)
for (const x of arr) count[x]++  // x 为负数时越界！

// ✅ 正确：减去最小值作为偏移
const min = Math.min(...arr)
const count = new Array(max - min + 1).fill(0)
for (const x of arr) count[x - min]++
```

## 实际应用

### 整数排序

当值域有限时，计数排序是最优选择。例如对考试成绩（0-100）排序、对年龄排序等。

### 字符串排序

基数排序可以用于字符串排序：从最后一个字符开始，逐字符进行稳定排序（MSD 或 LSD 方式）。这在后缀数组构建中非常有用。

### 后缀数组构建

后缀数组是字符串处理的重要工具。使用基数排序可以在 O(n log n) 时间内构建后缀数组，比比较排序更快。

### IP 地址排序

IP 地址可以看作 4 个 8 位整数，使用基数排序按每段排序，效率很高。

### 卡片排序机

基数排序的概念来自早期的卡片排序机（Hollerith 机器）。机器按卡片上的某一列孔洞分类，逐列处理，这正是 LSD 基数排序的物理实现。

## 总结

非比较排序通过绕过元素比较，突破了 O(n log n) 的理论下界：

- **计数排序**：最简单，适用于值域小的整数，O(n+k) 时间，O(n+k) 空间
- **基数排序**：逐位排序，适用于位数有限的整数，O(d·(n+k)) 时间
- **桶排序**：分桶处理，适用于均匀分布的数据，平均 O(n) 时间

选择非比较排序的关键条件：
1. **元素可表示为固定长度的数字或字符序列**
2. **值域范围 k 与元素数量 n 相比不会过大**
3. **需要稳定排序**

当数据满足这些条件时，非比较排序能提供显著的性能优势。理解这些算法也有助于深入理解排序的本质——排序不一定需要比较。
