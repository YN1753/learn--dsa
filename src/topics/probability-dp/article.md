# 概率DP (Probability DP)

## 概念解释

概率DP是一类特殊的动态规划问题，其状态转移方程涉及**概率**和**期望值**的计算。

与普通DP不同，概率DP的状态转移不是确定性的，而是基于某种随机过程。每个状态的值通常是：

- **概率**：到达某个状态的概率是多少？
- **期望**：从某个状态出发，到达目标状态的期望步数/收益是多少？

核心公式：

```
期望值 E[X] = Σ P(xi) × V(xi)
```

即：期望 = 每种可能结果的（概率 × 该结果的值）之和。

### 概率DP的两大类型

**类型一：求概率**

```
dp[i] = Σ dp[j] × P(j → i)
```

状态 i 的概率等于所有能转移到 i 的状态 j 的概率乘以从 j 到 i 的转移概率之和。

**类型二：求期望**

```
dp[i] = Σ P(i → j) × (cost(i, j) + dp[j])
```

从状态 i 出发的期望代价等于所有可能转移的（概率 × (转移代价 + 目标状态期望)）之和。

## 为什么重要

概率DP在算法竞赛和实际应用中都非常重要：

1. **骰子游戏分析**：计算各种棋盘游戏的最优策略和期望收益
2. **随机游走问题**：图上随机游走的期望步数、期望到达时间
3. **博弈论**：不完全信息博弈中的期望收益计算
4. **随机化算法**：分析随机化算法的期望时间复杂度
5. **机器学习**：隐马尔可夫模型（HMM）中的概率计算

## 核心原理

### 基本框架

概率DP的核心在于正确建模随机过程：

```typescript
// 求概率：dp[i] 表示到达状态 i 的概率
// 初始化：dp[start] = 1.0
// 转移：dp[i] += dp[j] × P(j → i)  （对所有能转移到 i 的 j）

// 求期望：dp[i] 表示从状态 i 到达目标的期望
// 通常需要逆向思考，从目标状态往回推
// 转移：dp[i] = Σ P(i → j) × (cost(i,j) + dp[j])
```

### 经典模型：骰子问题

掷一个 n 面骰子，求掷出所有面的期望次数。

```typescript
// dp[k] = 已经看到 k 个不同面，还需要掷多少次才能看到所有面的期望
// dp[n] = 0（已经看到所有面）
// dp[k] = k/n × dp[k] + (n-k)/n × dp[k+1] + 1
// 化简：dp[k] = dp[k+1] + n/(n-k)
function expectedDiceRolls(n: number): number {
  let expected = 0
  for (let k = n - 1; k >= 0; k--) {
    expected += n / (n - k)
  }
  return expected
}
```

### 经典模型：概率转移矩阵

在图上随机游走，从节点 u 到节点 v 的转移概率已知，求从起点到终点的期望步数。

```typescript
// 设 E[u] 为从节点 u 到达终点的期望步数
// E[终点] = 0
// E[u] = Σ P(u→v) × (1 + E[v])  对所有邻居 v
// 这会形成一个线性方程组，可以用高斯消元求解
```

### 常见技巧

1. **逆向思考**：期望DP通常需要从目标状态往回推
2. **方程展开**：当转移涉及自身时（dp[i] 出现在等式两边），需要移项化简
3. **高斯消元**：当状态转移形成环时，需要解线性方程组
4. **前缀和优化**：某些概率DP可以通过前缀和优化转移

## 常见错误

### 1. 概率之和不为1

```typescript
// 错误：转移概率没有正确归一化
// 例如骰子问题中，6面骰子每个面概率应为 1/6
// 如果概率之和不为1，结果就是错的

// 正确做法：确保每个状态的所有出边概率之和为 1
let totalProb = 0
for (const next of transitions) {
  totalProb += next.prob
}
// 检查：totalProb 应该等于 1（或接近 1，考虑浮点误差）
```

### 2. 忘记处理自环

```typescript
// 错误：当状态可以转移到自身时，直接忽略自环
dp[i] = prob_to_self × dp[i] + other_transitions + 1

// 正确做法：移项化简
// dp[i] = prob_to_self × dp[i] + S + 1
// dp[i] × (1 - prob_to_self) = S + 1
// dp[i] = (S + 1) / (1 - prob_to_self)
```

### 3. 混淆条件概率

```typescript
// 错误：把 P(A|B) 当成 P(A) 使用
// P(A|B) = P(A ∩ B) / P(B)

// 例如：已知事件 B 发生了，求 A 发生的概率
// 需要用条件概率，不能直接用无条件概率
```

### 4. 期望DP的方向搞反

```typescript
// 错误：求期望时正向DP
// 求概率可以正向（从起点推到终点）
// 求期望通常需要逆向（从终点往回推）

// 正确做法：
// E[终点] = 0
// for 状态 i 从终点附近往起点推:
//   E[i] = Σ P(i→j) × (cost + E[j])
```

### 5. 浮点精度问题

```typescript
// 错误：用 === 比较浮点数
if (prob === 0.5) { ... }  // 可能因精度问题失败

// 正确做法：使用误差范围
const EPS = 1e-9
if (Math.abs(prob - 0.5) < EPS) { ... }
```

## 实际应用

### 1. 骰子游戏（大富翁类问题）

在棋盘游戏中，玩家掷骰子前进，某些格子有特殊效果。概率DP可以计算：
- 到达每个格子的概率
- 通过某个格子的期望次数
- 游戏的期望时长

```typescript
// n 个格子的环形棋盘，掷 k 面骰子
// dp[i] = 到达格子 i 的概率
// dp[i] = Σ dp[j] × P(从 j 掷骰子到达 i)
```

### 2. 随机游走

在图上随机游走是概率DP的经典应用：

```typescript
// 无向图上从节点 s 出发，等概率走向邻居
// E[u] = 从节点 u 到达终点 t 的期望步数
// E[t] = 0
// E[u] = 1 + Σ E[v] / deg(u)  （对所有邻居 v）
```

### 3. 卡牌游戏

分析卡牌游戏中的最优策略：

```typescript
// 从牌堆中抽牌，已知剩余牌的情况
// 求抽到特定组合的期望抽牌次数
// dp[状态] = 当前手牌/剩余牌的情况下，还需要的期望次数
```

### 4. 蒙特卡洛方法验证

概率DP的结果可以用蒙特卡洛模拟来验证：

```typescript
function monteCarloSimulation(trials: number): number {
  let totalSteps = 0
  for (let t = 0; t < trials; t++) {
    // 模拟一次随机过程
    let steps = simulateOnce()
    totalSteps += steps
  }
  return totalSteps / trials  // 期望值的近似
}
```

### 5. 博弈论中的期望

在不完全信息博弈中，对手的策略可能包含随机性：

```typescript
// 混合策略纳什均衡
// dp[i] = 在状态 i 下，双方都按最优策略行动时的期望收益
```

## 总结

概率DP是动态规划与概率论的结合：

**核心概念**：
- 概率：某事件发生的可能性，所有可能结果的概率之和为 1
- 期望：随机变量的加权平均值，E[X] = Σ P(x) × x
- 条件概率：P(A|B) = P(A ∩ B) / P(B)

**解题框架**：
1. 定义状态：明确 dp[i] 代表什么（概率还是期望）
2. 建立转移：写出正确的概率转移方程
3. 处理自环：当转移涉及自身时，移项化简
4. 确定方向：求概率通常正向，求期望通常逆向
5. 处理精度：注意浮点数的精度问题

**常见模型**：
- 骰子问题：期望次数 = n × Hn（调和级数）
- 随机游走：期望步数需要解线性方程组
- 卡牌/棋盘游戏：状态转移 + 概率计算

概率DP的关键在于正确建模随机过程，理解概率和期望的数学本质，然后运用DP的框架进行高效计算。
