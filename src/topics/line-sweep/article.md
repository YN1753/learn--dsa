# 扫描线 (Line Sweep)

## 概念解释

**扫描线**是计算几何中一种强大而优雅的算法技巧。核心思想非常直观：想象有一条直线（扫描线），从平面的一侧缓缓移动到另一侧，在移动过程中处理遇到的"事件"。

最常用的扫描方向是**从左到右**（沿 x 轴正方向），此时扫描线是一条竖直的线。也可以从下到上扫描（沿 y 轴正方向），此时扫描线是水平的。

### 关键术语

| 术语 | 说明 |
|------|------|
| 扫描线 (Sweep Line) | 沿某个方向移动的直线，用于逐步处理几何元素 |
| 事件 (Event) | 扫描线到达某个关键位置时需要处理的操作 |
| 活跃集 (Active Set) | 当前扫描线位置上仍然"活跃"的几何元素集合 |
| 事件队列 (Event Queue) | 按扫描方向排序的所有事件，通常用优先队列维护 |

### 直观理解

想象你在看一本书，从第一页翻到最后一页。每翻到一页时，你会看到一些新的内容出现，一些旧的内容结束。扫描线算法就是这个过程的几何版本：

```
扫描线从左到右移动 →

    事件1      事件2      事件3
      |          |          |
      |  [===矩形A===]      |
      |     [===矩形B=====] |
      |          |          |
  ────┤          |          ├────
      |          |          |
```

每到一个事件位置，我们就更新当前的"活跃"信息，然后利用这些信息计算结果。

## 为什么重要

扫描线算法在计算几何中极为重要：

1. **矩形面积并**：求多个矩形覆盖的总面积，经典扫描线问题
2. **最近点对**：在平面点集中找距离最近的两个点
3. **线段交点**：找出一组线段所有的交点
4. **多边形面积**：计算复杂多边形的面积
5. **区间覆盖**：处理一维和二维的区间覆盖问题

扫描线的美在于它把复杂的二维问题转化为一系列简单的一维问题。

## 核心原理

### 算法框架

扫描线算法的通用框架如下：

```
1. 收集所有事件，按扫描方向排序
2. 初始化活跃集为空
3. 对每个事件：
   a. 计算上一个事件到当前事件的距离对答案的贡献
   b. 根据事件类型更新活跃集
   c. 重复直到所有事件处理完毕
```

### 经典应用：矩形面积并

给定 n 个与坐标轴平行的矩形，求它们覆盖的总面积。

**事件定义**：每个矩形产生两个事件：
- 左边界 (x1): 将矩形的 y 区间 [y1, y2] 加入活跃集
- 右边界 (x2): 将矩形的 y 区间 [y1, y2] 从活跃集移除

**面积计算**：每两个相邻事件之间，活跃集中的 y 区间总长度乘以 x 距离，就是这段的覆盖面积。

```typescript
interface Rectangle {
  x1: number; y1: number  // 左下角
  x2: number; y2: number  // 右上角
}

interface Event {
  x: number          // 事件的 x 坐标
  y1: number         // 矩形 y 区间下界
  y2: number         // 矩形 y 区间上界
  type: 'enter' | 'leave'  // 进入或离开
}

function rectangleAreaUnion(rects: Rectangle[]): number {
  // 1. 生成事件
  const events: Event[] = []
  for (const r of rects) {
    events.push({ x: r.x1, y1: r.y1, y2: r.y2, type: 'enter' })
    events.push({ x: r.x2, y1: r.y1, y2: r.y2, type: 'leave' })
  }

  // 2. 按 x 坐标排序
  events.sort((a, b) => a.x - b.x)

  // 3. 扫描
  let area = 0
  let prevX = 0
  const activeIntervals: { y1: number; y2: number; count: number }[] = []

  for (let i = 0; i < events.length; i++) {
    const e = events[i]

    // 计算上一段的面积贡献
    if (i > 0) {
      const dx = e.x - prevX
      const totalY = mergeIntervals(activeIntervals)
      area += dx * totalY
    }

    // 更新活跃集
    if (e.type === 'enter') {
      addInterval(activeIntervals, e.y1, e.y2)
    } else {
      removeInterval(activeIntervals, e.y1, e.y2)
    }

    prevX = e.x
  }

  return area
}
```

### 活跃集的维护

活跃集的核心操作是维护 y 方向的区间合并。常用方法：

**方法一：离散化 + 线段树**

将所有 y 坐标离散化，用线段树维护每个小区间被覆盖的次数和总覆盖长度。这是最高效的方法，单次更新 O(log n)。

**方法二：有序列表**

维护一个有序的区间列表，每次插入/删除时合并或拆分区间。实现简单但单次操作可能较慢。

### 时间复杂度

| 操作 | 复杂度 | 说明 |
|------|--------|------|
| 事件排序 | O(n log n) | 2n 个事件排序 |
| 离散化 | O(n log n) | 对 y 坐标排序去重 |
| 扫描过程 | O(n log n) | 每个事件在线段树上更新 |
| 总计 | O(n log n) | n 为矩形数量 |

## 常见错误

### 1. 同一 x 坐标事件的处理顺序

```typescript
// 错误：逐个事件处理，边更新边计算
for (const e of events) {
  area += (e.x - prevX) * currentCoverage  // 先算面积
  updateActiveSet(e)                         // 再更新
  prevX = e.x
}

// 正确：同一 x 的事件先全部更新，再算面积
// 或者：先算面积，再全部更新
const batches = groupByX(events)
for (const batch of batches) {
  area += (batch.x - prevX) * currentCoverage
  for (const e of batch) updateActiveSet(e)
  prevX = batch.x
}
```

### 2. 忘记离散化导致精度问题

```typescript
// 错误：直接用浮点数作为区间端点
// 浮点比较可能出错

// 正确：先收集所有不同的 y 坐标，排序后映射为整数索引
const yCoords = [...new Set(allYCoords)].sort((a, b) => a - b)
const yIndex = new Map(yCoords.map((y, i) => [y, i]))
```

### 3. 区间开闭处理不当

```typescript
// 错误：左闭右闭导致边界重叠时面积重复计算
[y1, y2]  // 如果两个矩形恰好相邻，边界会被计算两次

// 正确：注意区间的开闭定义，保持一致性
// 通常使用左闭右开 [y1, y2) 或根据具体问题调整
```

### 4. 活跃集更新与面积计算混在一起

扫描线的精髓是：**先用旧的活跃集计算面积，再用新的事件更新活跃集**。顺序不能颠倒。

## 实际应用

### 1. 游戏开发中的碰撞检测

在 2D 游戏中，扫描线可以高效检测哪些物体在水平方向上重叠，缩小碰撞检测的范围。

### 2. 地理信息系统 (GIS)

计算多个行政区域的覆盖面积，或多个建筑的投影面积并集。

### 3. 图像处理

像素级的区域覆盖计算，用于图像合成和遮罩处理。

### 4. 资源调度

在时间轴上用扫描线思想处理区间调度问题，如会议室占用、CPU 时间片分配等。

## 总结

扫描线是一种将二维问题降维为一维问题的强大技巧：

- **核心思想**：用一条移动的线逐步处理几何事件
- **关键步骤**：事件排序 -> 逐事件处理 -> 维护活跃集 -> 累积答案
- **经典应用**：矩形面积并、最近点对、线段交点检测
- **时间复杂度**：通常 O(n log n)，由排序和活跃集操作决定
- **实现要点**：注意同一坐标的事件批量处理、离散化避免精度问题、活跃集的正确维护

掌握扫描线思想，可以解决一大类计算几何问题。它也是理解更高级算法（如 Bentley-Ottmann 线段交点算法、Fortune 算法）的基础。
