# 计数排序 (Counting Sort)

## 概念解释

计数排序是一种**非比较排序**算法。它不通过元素之间的比较来确定顺序，而是利用数组下标直接统计每个值出现的次数，然后依次输出。

核心思想非常直观：如果你知道数组中有多少个元素小于等于某个值，你就知道这个值应该放在哪个位置。

### 算法步骤

计数排序分为三个阶段：

1. **统计计数**：遍历输入数组，统计每个值出现的次数，存入计数数组 `count[]`
2. **计算前缀和**：将计数数组转换为前缀和数组，`count[i]` 表示值小于等于 `i` 的元素个数
3. **反向放置**：从后往前遍历输入数组，根据前缀和确定每个元素的输出位置，放置到输出数组中

```
输入: [4, 2, 2, 8, 3, 3, 1]

第1步 - 统计计数:
  值:    1  2  3  4  5  6  7  8
  次数:  1  2  2  1  0  0  0  1

第2步 - 前缀和:
  累积:  1  3  5  6  6  6  6  7
  (含义: 值 ≤ x 的元素共有 prefixSum[x] 个)

第3步 - 反向放置:
  从后往前遍历原数组，每次将元素放到 count[val]-1 的位置
  然后 count[val]--

输出: [1, 2, 2, 3, 3, 4, 8]
```

### 为什么是反向放置？

反向放置是为了保证**稳定性**——相等元素在输出中的相对顺序与输入中相同。如果正向放置，后出现的相同值会被放到前面，破坏稳定性。

## 为什么重要

### 突破 O(n log n) 下界

基于比较的排序算法，理论最优时间复杂度为 O(n log n)。计数排序通过"不比较"的方式绕过了这个限制，达到 O(n+k) 的线性时间。

### 基数排序的基础

基数排序的核心子程序就是计数排序。每一位的排序都使用计数排序来完成，保证了稳定性和效率。

### 实际场景中的高效选择

当值域 k 远小于元素个数 n 时（例如对 100 万个 [0, 1000] 范围内的整数排序），计数排序比任何比较排序都快得多。

### 稳定性的天然保证

计数排序是稳定排序，这在多关键字排序中非常重要。例如先按成绩排序、再按姓名排序时，稳定性保证了同名学生的成绩顺序不会被破坏。

## 核心原理

### 详细推导

假设输入数组为 `arr[0..n-1]`，值域为 `[minVal, maxVal]`，令 `k = maxVal - minVal + 1`。

**第 1 步：统计计数**

创建大小为 k 的计数数组 `count[0..k-1]`，初始全为 0。遍历输入数组：

```typescript
for (let i = 0; i < n; i++) {
  count[arr[i] - minVal]++
}
```

时间复杂度：O(n)

**第 2 步：计算前缀和**

```typescript
for (let i = 1; i < k; i++) {
  count[i] += count[i - 1]
}
```

转换后，`count[v]` 表示值 ≤ `v + minVal` 的元素个数，也就是值 `v + minVal` 应该放置的末尾位置+1。

时间复杂度：O(k)

**第 3 步：反向放置**

```typescript
const output = new Array(n)
for (let i = n - 1; i >= 0; i--) {
  const val = arr[i]
  output[--count[val - minVal]] = val
}
```

从后往前遍历，保证稳定性。`--count[val - minVal]` 先自减再作为索引，因为 count 存的是"末尾位置+1"。

时间复杂度：O(n)

### 复杂度分析

| 指标 | 复杂度 |
|------|--------|
| 时间（所有情况） | O(n + k) |
| 空间 | O(n + k) |
| 稳定性 | 稳定 |

其中 n 是元素个数，k 是值域大小（maxVal - minVal + 1）。

### 适用条件

- **值域较小**：k 不能太大，否则计数数组占用过多空间
- **整数或可映射为整数的数据**：需要能够将数据映射到连续的整数下标
- **需要稳定排序时**：计数排序天然稳定

### 不适用的场景

- **值域很大**：如对 [0, 10^9] 范围内的 100 个数排序，k = 10^9，空间和时间都无法接受
- **浮点数排序**：浮点数无法直接映射为数组下标
- **数据分布极度稀疏**：大量空间被浪费

## 常见错误

### 1. 忘记处理值域偏移

```typescript
// 错误：假设值从 0 开始
count[arr[i]]++  // 如果 arr[i] > count.length 会越界！

// 正确：减去最小值作为偏移
count[arr[i] - minVal]++
```

### 2. 正向放置导致不稳定

```typescript
// 错误：正向遍历放置
for (let i = 0; i < n; i++) {
  output[count[arr[i] - minVal] - 1] = arr[i]
  count[arr[i] - minVal]--
}
// 相同值的元素会被反序放置

// 正确：反向遍历放置
for (let i = n - 1; i >= 0; i--) {
  output[--count[arr[i] - minVal]] = arr[i]
}
```

### 3. 前缀和含义理解错误

```typescript
// 前缀和的含义是：值 ≤ (i + minVal) 的元素共有 count[i] 个
// 因此 count[i] 也是值 (i + minVal) 放置时的"末尾位置+1"
// 放置时需要先 --count[i] 再使用
```

### 4. 值域过大导致内存溢出

```typescript
// 错误：对 [0, 10^9] 范围的数据使用计数排序
const count = new Array(1e9 + 1)  // 内存爆炸！

// 正确：先检查值域大小
const k = maxVal - minVal + 1
if (k > 1e7) {
  // 改用其他排序算法
}
```

## 实际应用

### 基数排序的子程序

基数排序对每一位进行排序时，使用的就是计数排序。由于计数排序是稳定的，保证了高位排序不会破坏低位的顺序。

### 桶排序的简化版

当数据可以映射为小范围整数时，计数排序就是最简单的"桶排序"——每个值一个桶。

### 频率统计

计数排序的第一步（统计频率）在很多场景中单独使用：统计字符出现次数、词频统计、直方图生成等。

### 竞赛编程

在算法竞赛中，当值域较小时，计数排序是实现简单、效率极高的选择。常用于处理离散化后的数据。

### 学生成绩排序

对百分制成绩（0-100）排序是计数排序的经典应用场景：101 个桶就够了，时间复杂度 O(n)，远优于 O(n log n)。

## 总结

计数排序是一种简单而高效的非比较排序算法：

- **核心思想**：统计每个值出现的次数，利用前缀和确定放置位置
- **时间复杂度**：O(n + k)，当 k = O(n) 时为线性
- **空间复杂度**：O(n + k)
- **稳定性**：稳定（反向放置保证）
- **适用场景**：值域较小的整数排序、作为基数排序的子程序
- **局限性**：值域大时空间和时间都不划算，不能直接处理浮点数

理解计数排序，是理解"为什么比较排序有 O(n log n) 下界"以及"如何突破这个下界"的关键一步。
