# A* 搜索算法 (A* Search)

## 概念解释

**A* 搜索算法**是一种用于在加权图中寻找从起点到目标节点**最短路径**的启发式搜索算法。它由 Peter Hart、Nils Nilsson 和 Bertram Raphael 于 1968 年提出，是目前应用最广泛的路径规划算法之一。

### 核心公式

A* 算法的核心是评估函数：

```
f(n) = g(n) + h(n)
```

其中：

- **f(n)**：节点 n 的综合评估值，表示经过节点 n 到达目标的估计总代价
- **g(n)**：从起点到节点 n 的**实际代价**（已经走过的路径长度，是确定的）
- **h(n)**：从节点 n 到目标节点的**启发式估计代价**（尚未走的路径，需要估计）

### 可采纳启发式 (Admissible Heuristic)

A* 算法要求启发式函数 h(n) 必须是**可采纳的**（Admissible），即对于所有节点 n：

```
h(n) <= h*(n)
```

其中 h*(n) 是从 n 到目标的真实最短路径代价。换句话说，启发式函数**永远不能高估**到达目标的代价。这个性质保证了 A* 能找到最优解。

### 常见启发式函数

**曼哈顿距离**（适用于只能上下左右移动的网格）：
```
h(n) = |n.x - goal.x| + |n.y - goal.y|
```

**欧几里得距离**（适用于可以任意方向移动的场景）：
```
h(n) = sqrt((n.x - goal.x)² + (n.y - goal.y)²)
```

**切比雪夫距离**（适用于可以对角移动的网格）：
```
h(n) = max(|n.x - goal.x|, |n.y - goal.y|)
```

## 为什么重要

A* 算法在计算机科学和工程领域具有极其重要的地位：

- **最优且完备**：当使用可采纳启发式时，A* 保证找到最短路径（最优性），并且在有限图中一定能找到解（完备性）
- **比 Dijkstra 更高效**：Dijkstra 算法可以看作 h(n)=0 的 A*，它向所有方向均匀扩展。A* 通过启发式函数引导搜索方向，在单对节点的最短路径问题中通常探索更少的节点
- **灵活的启发式设计**：不同的启发式函数适用于不同的场景，可以在效率和最优性之间做权衡
- **广泛的实际应用**：从游戏 AI 到机器人导航，从 GPS 路径规划到拼图求解，A* 无处不在

### 与其他搜索算法的比较

| 特性 | A* | Dijkstra | 贪心最佳优先 | BFS |
|------|-----|----------|------------|-----|
| 使用启发式 | 是 (g+h) | 否 (g) | 是 (h) | 否 |
| 保证最优 | 是 | 是 | 否 | 是(等权) |
| 时间复杂度 | O(b^d) | O(V² 或 E log V) | O(b^d) | O(V+E) |
| 搜索方向 | 有目标导向 | 均匀扩展 | 强目标导向 | 均匀扩展 |
| 适用场景 | 带权图单对最短路径 | 带权图单源最短路径 | 快速找到可行解 | 等权图最短路径 |

其中 b 为分支因子，d 为解的深度。

## 核心原理

### 数据结构

A* 算法维护两个关键集合：

**开放列表 (Open List)**：一个优先队列，存储所有已发现但尚未扩展的节点。每次取出 f 值最小的节点进行扩展。通常使用最小堆实现。

**关闭列表 (Closed List)**：一个集合，存储已经扩展过的节点。目的是避免重复处理同一个节点。

### 算法步骤

1. 将起点加入开放列表，g(start)=0，f(start)=h(start)
2. 重复以下步骤直到找到目标或开放列表为空：
   - 从开放列表中取出 f 值最小的节点 current
   - 如果 current 是目标节点，重建路径并返回
   - 将 current 加入关闭列表
   - 对 current 的每个邻居 neighbor：
     - 如果 neighbor 在关闭列表中，跳过
     - 计算经过 current 到达 neighbor 的新代价 newG = g(current) + cost(current, neighbor)
     - 如果 neighbor 不在开放列表中，将其加入
     - 如果 newG < g(neighbor)，更新 g(neighbor) = newG，设置 parent(neighbor) = current
3. 如果开放列表为空且未找到目标，说明不存在路径

### 伪代码

```typescript
function aStar(graph, start, goal, heuristic) {
  const openList = new MinHeap()     // 按 f 值排序的优先队列
  const closedSet = new Set()        // 已扩展的节点集合

  const g = new Map()                // 从起点到各节点的实际代价
  const parent = new Map()           // 用于路径重建的前驱节点
  g.set(start, 0)

  openList.push({ node: start, f: heuristic(start, goal) })

  while (!openList.isEmpty()) {
    const current = openList.pop().node

    if (current === goal) {
      return reconstructPath(parent, goal)  // 找到目标，重建路径
    }

    closedSet.add(current)

    for (const [neighbor, cost] of graph.neighbors(current)) {
      if (closedSet.has(neighbor)) continue

      const newG = g.get(current) + cost

      if (!g.has(neighbor) || newG < g.get(neighbor)) {
        g.set(neighbor, newG)
        parent.set(neighbor, current)
        const f = newG + heuristic(neighbor, goal)
        openList.push({ node: neighbor, f })
      }
    }
  }

  return null  // 无路径
}
```

### 执行过程示例

在一个 5x5 的网格中，起点为左上角 (0,0)，目标为右下角 (4,4)，中间有障碍物：

```
S . . . .
. # # . .
. . . . .
. # # # .
. . . . G
```

使用曼哈顿距离作为启发式函数。

**初始化**：起点 (0,0) 加入开放列表，g=0, h=8, f=8

**第 1 步**：取出 (0,0)，扩展邻居 (0,1) 和 (1,0)
- (0,1): g=1, h=7, f=8
- (1,0): g=1, h=7, f=8

**第 2 步**：取出 f 最小的节点（假设取 (0,1)），扩展邻居 (0,2)
- (0,2): g=2, h=6, f=8

...以此类推，A* 会优先沿着靠近目标的方向搜索，遇到障碍物时自动绕行。

### 时间与空间复杂度

| 指标 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(b^d) | b 为分支因子，d 为最优解深度 |
| 空间 | O(b^d) | 需要存储所有已生成的节点 |

在网格图中，若网格大小为 N×N，时间复杂度为 O(N² log N)。启发式函数的质量直接影响实际性能——启发式越接近真实代价，需要探索的节点越少。

## 常见错误

### 1. 使用不可采纳的启发式

**错误表现**：启发式函数高估了到达目标的代价，导致 A* 找到的不是最短路径。

```typescript
// 错误：使用了 3 倍曼哈顿距离，会高估实际代价
function badHeuristic(node, goal) {
  return 3 * (Math.abs(node.x - goal.x) + Math.abs(node.y - goal.y))
}
```

**正确做法**：确保 h(n) 永远不超过真实最短路径代价。
```typescript
// 正确：使用 1 倍曼哈顿距离（对于只能上下左右移动的网格）
function goodHeuristic(node, goal) {
  return Math.abs(node.x - goal.x) + Math.abs(node.y - goal.y)
}
```

### 2. 混淆 A* 与贪心最佳优先搜索

**错误**：只使用 h(n) 作为评估函数，忽略 g(n)。

```typescript
// 错误：这是贪心最佳优先搜索，不是 A*！
const f = h(neighbor, goal)  // 只考虑启发式，忽略已走过的路径
```

**区别**：贪心最佳优先搜索只看 h(n)，速度快但不保证最优。A* 同时考虑 g(n) 和 h(n)，保证最优。

### 3. 未更新开放列表中已有节点的 g 值

**错误表现**：当发现到达某个已在开放列表中的节点有更短路径时，没有更新其 g 值和父节点。

```typescript
// 错误：发现更短路径却不更新
if (!openSet.has(neighbor)) {
  openSet.add(neighbor)
  g.set(neighbor, newG)
  // 但如果 neighbor 已在 openSet 中且 newG < g.get(neighbor) 呢？
}
```

**正确做法**：
```typescript
if (!g.has(neighbor) || newG < g.get(neighbor)) {
  g.set(neighbor, newG)
  parent.set(neighbor, current)
  if (!openSet.has(neighbor)) {
    openSet.add(neighbor)
  }
  // 更新优先队列中的 f 值
}
```

### 4. 在不可行场景下无限循环

**错误**：没有检查开放列表为空的情况，或者在有障碍物的图中没有正确判断「无解」。

**正确做法**：当开放列表为空时，说明所有可达节点都已探索完毕，若未找到目标则返回「无路径」。

### 5. 启发式函数与移动方式不匹配

**错误**：在可以对角移动的网格中使用曼哈顿距离，会导致 h(n) 高估（不可采纳），因为对角移动的实际最短距离小于曼哈顿距离。

**正确做法**：对角移动时使用切比雪夫距离或欧几里得距离。

## 实际应用

### 1. 游戏路径规划

A* 是游戏开发中最常用的路径规划算法。在 RTS（即时战略）游戏中，单位需要在地图上找到从当前位置到目标位置的最短路径，同时避开障碍物和其他单位。

```typescript

  const heuristic = (a: Pos, b: Pos) =>
    Math.abs(a.x - b.x) + Math.abs(a.y - b.y)  // 曼哈顿距离

  // ... A* 主循环 ...
}
```

### 2. GPS 导航系统

GPS 导航使用 A* 或其变体在道路网络中寻找最短或最快路线。启发式函数可以是直线距离除以最大速度限制。

### 3. 机器人运动规划

在机器人学中，A* 用于在已知环境中规划从起点到目标点的无碰撞路径。配置空间（Configuration Space）中的每个点代表机器人的一个姿态。

### 4. 拼图求解

八数码问题、十五数码问题等滑动拼图可以用 A* 求解。启发式函数通常使用「不在位方块数」或「曼哈顿距离之和」。

### 5. 网络路由

在计算机网络中，A* 可以用于寻找考虑延迟、带宽等多个因素的最优路由路径。

## 总结

A* 搜索算法是路径规划领域最重要的算法之一，它巧妙地结合了 Dijkstra 算法的最优性和贪心搜索的高效性。

**关键要点**：
- 评估函数 f(n) = g(n) + h(n)，g 是实际代价，h 是启发式估计
- 启发式函数必须是可采纳的（不高于真实代价），才能保证最优性
- 使用开放列表（优先队列）和关闭列表管理节点
- A* 是完备的（有限图中一定能找到解，如果存在）和最优的（使用可采纳启发式）
- 时间和空间复杂度为 O(b^d)，实际性能取决于启发式函数的质量

**适用场景**：
- 带权图中的单对最短路径问题
- 游戏 AI 路径规划
- 机器人导航与运动规划
- 拼图和谜题求解

**局限性**：
- 需要设计合适的启发式函数
- 在大规模图中空间开销较大（需要存储所有已生成节点）
- 对于单源多目标问题，Dijkstra 可能更合适

掌握 A* 算法，你将拥有解决路径规划问题的核心工具。理解启发式函数的设计原则，能帮助你在不同场景下做出最优选择。
