# 桶排序 (Bucket Sort)

## 概念解释

桶排序是一种**非比较排序**算法，核心思想是将待排序的元素分配到有限数量的"桶"中，每个桶内再分别排序（可以使用其他排序算法或递归地使用桶排序），最后按桶的顺序依次取出所有元素，得到有序序列。

想象你有一堆不同重量的水果，你要按重量排序。你可以准备几个箱子，第一个箱子放 0-100g 的，第二个放 100-200g 的，以此类推。每个箱子里的水果再单独称重排序，最后把所有箱子里的水果按顺序取出来——这就是桶排序的过程。

```
原数组: [0.42, 0.32, 0.23, 0.52, 0.25, 0.47, 0.51]

桶[0.0-0.2]:  (空)
桶[0.2-0.4]:  [0.32, 0.23, 0.25] → 排序 → [0.23, 0.25, 0.32]
桶[0.4-0.6]:  [0.42, 0.52, 0.47, 0.51] → 排序 → [0.42, 0.47, 0.51, 0.52]

合并: [0.23, 0.25, 0.32, 0.42, 0.47, 0.51, 0.52]
```

## 为什么重要

### 平均 O(n) 时间复杂度

当数据在值域上均匀分布时，桶排序可以达到 **O(n)** 的平均时间复杂度。这突破了比较排序的 O(n log n) 下界（虽然桶排序是非比较排序，不违反决策树下界）。

### 计数排序的泛化

桶排序可以看作计数排序的泛化。计数排序为每个值创建一个计数器（相当于每个值一个桶），而桶排序将值域划分为若干区间，每个区间对应一个桶。当桶的数量等于值域大小时，桶排序退化为计数排序。

### 处理浮点数的天然优势

与计数排序和基数排序主要处理整数不同，桶排序天然适合处理浮点数——只需将浮点数映射到对应的桶即可。

### 并行化潜力

桶排序的分配阶段和各桶的排序阶段可以并行执行。在多核处理器或分布式系统中，每个桶可以独立排序，最后合并结果。

## 核心原理

### 算法步骤

1. **确定桶的数量和范围**：根据数据的值域，将其划分为 k 个等宽的区间（桶）
2. **分配元素**：遍历原数组，将每个元素放入对应的桶中
3. **桶内排序**：对每个桶内的元素单独排序（常用插入排序，因为桶内元素通常较少）
4. **合并结果**：按桶的顺序，依次将所有桶中的元素取出，拼接成最终的有序数组

```
原数组: [29, 25, 3, 49, 9, 37, 21, 43]

桶数量 = 5, 每桶范围 = 10

桶[0-9]:    [3, 9]
桶[10-19]:  (空)
桶[20-29]:  [29, 25, 21]
桶[30-39]:  [37]
桶[40-49]:  [49, 43]

桶内排序:
  桶[0-9]:    [3, 9]
  桶[20-29]:  [21, 25, 29]
  桶[30-39]:  [37]
  桶[40-49]:  [43, 49]

合并: [3, 9, 21, 25, 29, 37, 43, 49]
```

### 桶数量的选择

桶数量 k 的选择直接影响排序性能：

- **k 太小**：每个桶中的元素过多，桶内排序代价大
- **k 太大**：桶的数量接近数据量，空间浪费，且大量桶可能为空
- **经验法则**：通常取 k ≈ n（数据量），或者根据数据分布特征选择

对于均匀分布在 [0, 1) 的浮点数，通常取 k = n，每个桶覆盖区间宽度为 1/n。

### 复杂度分析

| 情况 | 时间复杂度 | 说明 |
|------|-----------|------|
| 最好 | O(n) | 数据完全均匀分布，每个桶 1 个元素 |
| 平均 | O(n + n²/k + k) | k 为桶数量，n²/k 为桶内排序总代价 |
| 最坏 | O(n²) | 所有元素落入同一个桶 |

当 k = O(n) 且数据均匀分布时，平均复杂度简化为 O(n)。

空间复杂度：O(n + k)，需要 n 个元素的空间和 k 个桶的元数据。

稳定性：取决于桶内使用的排序算法。如果桶内使用插入排序等稳定排序，则桶排序整体是稳定的。

## 可视化说明

在可视化面板中，你可以直观地观察桶排序的三个阶段：

- **分配阶段**：元素从原数组上方依次飞入对应的桶中，不同桶用不同颜色标识
- **排序阶段**：每个桶内的元素独立进行排序动画（使用插入排序）
- **收集阶段**：排序后的元素从各桶中依次取出，拼接成最终的有序数组

颜色说明：
- 蓝色表示待分配的元素
- 不同颜色的桶背景表示不同的桶
- 黄色高亮当前正在处理的元素
- 绿色表示已排好序的元素

你可以通过控制栏调整数组大小、桶的数量和播放速度，还可以自定义输入数据，观察不同分布对排序过程的影响。

## 常见错误

### 1. 桶数量选择不当

```typescript
// ❌ 错误：桶数量固定为 5，不管数据量多少
const bucketCount = 5
// 当 n=10000 时，每个桶平均 2000 个元素，桶内排序代价巨大

// ✅ 正确：根据数据量动态选择桶数量
const bucketCount = Math.max(1, Math.floor(arr.length / 5))
```

### 2. 未处理空桶

```typescript
// ❌ 错误：假设每个桶都有元素
for (let i = 0; i < bucketCount; i++) {
  result.push(...insertionSort(buckets[i]))
  // 当 buckets[i] 为空时不会出错，但浪费了排序调用
}

// ✅ 正确：检查桶是否为空
for (let i = 0; i < bucketCount; i++) {
  if (buckets[i].length > 0) {
    insertionSort(buckets[i])
    result.push(...buckets[i])
  }
}
```

### 3. 假设桶排序总是 O(n)

```typescript
// ❌ 错误的期望：桶排序总是比快排快
// 实际：当数据分布极不均匀时，桶排序可能退化为 O(n²)
// 例如：[1, 1, 1, 1, 1, 1, 100] —— 几乎所有元素落入同一个桶

// ✅ 正确：理解桶排序的 O(n) 依赖于均匀分布假设
// 如果数据分布未知，应优先考虑快速排序等通用排序
```

### 4. 浮点数桶索引计算错误

```typescript
// ❌ 错误：浮点数桶索引可能越界
const bucketIndex = Math.floor(value * bucketCount)
// 当 value = 1.0 时，bucketIndex = bucketCount，越界！

// ✅ 正确：限制桶索引范围
const bucketIndex = Math.min(
  Math.floor(value * bucketCount),
  bucketCount - 1
)
```

## 实际应用

### 浮点数排序

桶排序最经典的应用场景是对均匀分布的浮点数排序。在 [0, 1) 区间的浮点数可以直接乘以桶数量确定桶索引，非常高效。

### 外部排序的组件

在处理超出内存容量的大规模数据时，桶排序的思想被用于外部排序：将数据分块写入磁盘（类似分配到桶），每块在内存中排序后归并。

### 直方图均衡化

图像处理中的直方图均衡化本质上是桶排序思想的应用——将像素值分配到不同的灰度级桶中，重新映射以增强对比度。

### 数据分片

分布式系统中，桶排序的分配思想被用于数据分片（Sharding）：根据数据的某个属性值，将数据分配到不同的节点或分区中处理。

### 近似排序

当不需要精确排序，只需要将数据大致分组时（如将学生成绩分为 A/B/C/D 等级），桶排序非常高效。

## 总结

桶排序是一种简单而高效的非比较排序算法：

- **核心思想**：将数据分配到多个桶中，各桶独立排序后合并
- **时间复杂度**：平均 O(n)（数据均匀分布时），最坏 O(n²)（所有元素在一个桶中）
- **空间复杂度**：O(n + k)，k 为桶数量
- **稳定性**：取决于桶内使用的排序算法
- **适用场景**：数据均匀分布在已知范围内

选择桶排序的关键条件：
1. **数据分布已知或可预估**——均匀分布时性能最优
2. **值域范围合理**——桶数量可控
3. **对浮点数排序**——桶排序天然支持

当数据分布不均匀或未知时，快速排序等通用排序算法是更安全的选择。
