# 快速幂 (Binary Exponentiation)

## 什么是快速幂？

快速幂（Binary Exponentiation，也称 Exponentiation by Squaring）是一种在 **O(log n)** 时间内计算 **a^n mod m** 的算法。

朴素方法计算 a^n 需要 n-1 次乘法，时间复杂度为 O(n)。当 n 很大时（例如 n = 10^18），朴素方法完全不可行。快速幂利用指数的二进制表示，将问题分解为若干次平方和乘法，仅需约 log₂(n) 步即可完成计算。

## 为什么重要？

快速幂是计算机科学中最基础、最常用的算法之一，广泛应用于以下场景：

1. **RSA 加密算法**：RSA 的核心操作就是大数的模幂运算。加密和解密都需要计算 m^e mod n，其中 e 和 n 可以是数百位的大数。没有快速幂，现代加密体系将无法工作。

2. **竞赛编程**：几乎所有涉及取模的题目都会用到快速幂。例如计算组合数 C(n, k) mod p、矩阵快速幂求斐波那契数列第 n 项等。

3. **矩阵快速幂**：将快速幂的思想推广到矩阵乘法，可以在 O(k³ log n) 时间内计算矩阵的 n 次幂（k 为矩阵维度），用于求解线性递推关系。

4. **数论计算**：欧拉定理、中国剩余定理、原根判定等数论算法中，快速幂是不可或缺的基础工具。

5. **密码学**：Diffie-Hellman 密钥交换、ElGamal 加密等协议都依赖模幂运算。

## 核心原理

### 二进制分解

快速幂的核心思想是将指数 n 用二进制表示。例如：

```
10 的二进制 = 1010₂
```

这意味着：

```
2^10 = 2^(8+2) = 2^8 × 2^2
```

更一般地，对于任意 n：

```
a^n = a^(b_k × 2^k + ... + b_1 × 2 + b_0)
    = (a^(2^k))^b_k × ... × (a^2)^b_1 × a^b_0
```

其中 b_i 是 n 的二进制表示的第 i 位（0 或 1）。

### 算法过程

从低位到高位遍历 n 的每一位：

1. 如果当前位为 1，则将当前的底数乘入结果
2. 每一步都将底数平方（a → a² → a⁴ → a⁸ → ...）
3. 每次乘法后都取模，防止溢出

伪代码：

```
function power(base, exp, mod):
    result = 1
    base = base mod mod
    while exp > 0:
        if exp is odd:           // 当前最低位为 1
            result = (result * base) mod mod
        exp = exp >> 1           // 右移一位
        base = (base * base) mod mod  // 底数平方
    return result
```

### 计算 2^10 mod 1000 的过程

| 步骤 | exp | exp (二进制) | 最低位 | 操作 | base | result |
|------|-----|-------------|--------|------|------|--------|
| 初始 | 10 | 1010 | - | - | 2 | 1 |
| 1 | 10 | 1010 | 0 (偶) | 跳过 | 4 | 1 |
| 2 | 5 | 101 | 1 (奇) | result = 1 × 4 | 16 | 4 |
| 3 | 2 | 10 | 0 (偶) | 跳过 | 256 | 4 |
| 4 | 1 | 1 | 1 (奇) | result = 4 × 256 | 65536 | 1024 mod 1000 = 24 |

最终结果：**2^10 mod 1000 = 24**（2^10 = 1024，1024 mod 1000 = 24）

算法将 4 次迭代（对应 4 个二进制位）中的 2 次乘法合并为最终结果，总共只需 2 次乘法（result 层面），而非朴素方法的 9 次。

### 时间复杂度分析

- 每次循环 exp 至少减半（右移一位）
- 循环次数 = ⌊log₂(n)⌋ + 1
- 每次循环做 O(1) 次乘法和取模
- **总时间复杂度：O(log n)**

### 空间复杂度

- 只需要常数个变量
- **空间复杂度：O(1)**

## 常见错误

### 1. 忘记每步取模

这是最常见的错误。如果只在最后取模，中间结果可能溢出（即使是 64 位整数也会溢出）。

```
// 错误
result = (result * base) % mod  // 只在最后取模

// 正确
result = ((result % mod) * (base % mod)) % mod  // 每步取模
```

### 2. 整数溢出

即使每步取模，两个接近 mod 的数相乘仍可能溢出。对于 mod 接近 10^9 的情况，需要使用 `BigInt` 或在乘法时先取模。

```typescript
// 可能溢出 (当 mod 接近 2^31 时)
const mul = (a: number, b: number, mod: number): number => {
  return ((a % mod) * (b % mod)) % mod
}

// 安全写法：使用 BigInt
const safeMul = (a: bigint, b: bigint, mod: bigint): bigint => {
  return (a * b) % mod
}
```

### 3. 指数为 0 的特殊情况

任何数的 0 次方等于 1。如果算法没有正确处理这个边界情况，可能返回错误结果。

### 4. 负指数处理

标准快速幂不处理负指数。如果需要计算 a^(-1) mod m，需要使用扩展欧几里得算法或费马小定理求模逆元。

### 5. 模数为 1 的特殊情况

任何数 mod 1 = 0，应该直接返回 0。

## 实际应用

### 1. 计算组合数取模

```
C(n, k) mod p = n! / (k! × (n-k)!) mod p
```

除法取模需要用到模逆元，而模逆元的计算（费马小定理方法）需要快速幂。

### 2. 矩阵快速幂求斐波那契数列

斐波那契数列可以用矩阵表示：

```
| F(n+1) |   | 1  1 |^n   | 1 |
| F(n)   | = | 1  0 |   × | 0 |
```

用矩阵快速幂可以在 O(log n) 时间内求出 F(n)。

### 3. 大数运算

在需要计算 a^b（b 非常大）的场景中，快速幂是唯一可行的方法。例如：

- 密码学中的模幂运算
- 竞赛中的大数取模
- 数论函数的计算

### 4. 快速幂取模在哈希中的应用

某些字符串哈希算法中需要计算 p^i mod M，快速幂可以高效完成这个计算。

## 总结

快速幂是一种简单而强大的算法：

- **核心思想**：利用指数的二进制分解，将 O(n) 的乘法次数降低到 O(log n)
- **关键操作**：平方 + 按位判断 + 取模
- **时间复杂度**：O(log n)
- **空间复杂度**：O(1)
- **适用场景**：大数模幂运算、密码学、竞赛编程、矩阵快速幂

掌握快速幂是学习数论和密码学的第一步。它的思想——将问题分解为二进制位——在计算机科学的许多领域都有应用。
