# 三分搜索 (Ternary Search)

## 概念解释

三分搜索（Ternary Search）是一种在**单峰函数**上寻找极值（最大值或最小值）的搜索算法。它的核心思想是：每次将搜索区间**三等分**为三段，通过比较两个分点的函数值，确定极值在哪一段内，从而排除掉区间的三分之一。

想象你在一座只有一个山顶的山脉上，你站在山脚的两个不同位置测量海拔。如果左边的海拔比右边高，说明山顶一定在你左边位置的右侧（包含右边）；反之则在右边位置的左侧（包含左边）。这样你就能排除掉一部分区域，缩小搜索范围。这就是三分搜索的本质。

### 什么是单峰函数？

单峰函数是指在给定区间上**只有一个极值点**的函数：

- **单峰（Unimodal）**：先增后减，只有一个极大值
- **单谷（Univalley）**：先减后增，只有一个极小值

```
单峰函数示例：          单谷函数示例：

    *                      *
   * *                    * *
  *   *                  *   *
 *     *                *     *
*       *              *       *
```

### 与二分搜索的区别

| 特性 | 二分搜索 | 三分搜索 |
|------|---------|---------|
| 适用对象 | 有序数组 | 单峰函数 |
| 每次排除 | 1/2 区间 | 1/3 区间 |
| 比较次数 | 1 次 | 2 次 |
| 时间复杂度 | O(log n) | O(log_{1.5} n) |
| 应用场景 | 查找值 | 求极值 |

## 为什么重要

### 1. 连续域优化的基础工具

很多实际问题可以归结为在一个连续区间上求单峰函数的极值。三分搜索提供了一种简单而有效的数值方法，不需要计算导数就能找到极值点。

### 2. 竞赛中的高频技巧

在算法竞赛中，三分搜索经常与几何问题、贪心策略结合使用。例如：

- 凸包上的最远点对
- 抛物线上的最优位置
- 距离函数的最小值

### 3. 替代黄金分割搜索

三分搜索是黄金分割搜索的一种简化版本。虽然理论上黄金分割搜索的效率略高（减少一次函数求值），但三分搜索的实现更直观，且在实际应用中差异不大。

### 4. 与凸优化的联系

对于凸函数（或凹函数），三分搜索保证能找到全局最优解。这使得它在简单凸优化问题中非常实用。

## 核心原理

### 浮点三分搜索

浮点三分搜索适用于连续定义域上的函数：

```typescript
function ternarySearch(
  left: number,
  right: number,
  f: (x: number) => number,
  epsilon: number = 1e-9
): number {
  while (right - left > epsilon) {
    const m1 = left + (right - left) / 3
    const m2 = right - (right - left) / 3
    const f1 = f(m1)
    const f2 = f(m2)

    if (f1 < f2) {
      left = m1    // 极大值在右侧
    } else {
      right = m2   // 极大值在左侧
    }
  }

  return (left + right) / 2
}
```

### 执行过程详解

以函数 `f(x) = -(x-3)^2 + 10` 在区间 `[0, 6]` 上找最大值为例：

```
第 1 步: [0, 6]
  m1 = 2, f(2) = 9
  m2 = 4, f(4) = 9
  f(m1) == f(m2) → 排除 [4, 6]，区间变为 [0, 4]

第 2 步: [0, 4]
  m1 = 1.333, f(1.333) = 7.222
  m2 = 2.667, f(2.667) = 9.889
  f(m1) < f(m2) → 排除 [0, 1.333]，区间变为 [1.333, 4]

第 3 步: [1.333, 4]
  m1 = 2.222, f(2.222) = 9.391
  m2 = 3.111, f(3.111) = 9.988
  f(m1) < f(m2) → 排除 [1.333, 2.222]，区间变为 [2.222, 4]

... 逐步收敛到 x ≈ 3.0
```

### 整数三分搜索

当定义域是整数时，三分搜索略有不同：

```typescript
function integerTernarySearch(
  left: number,
  right: number,
  f: (x: number) => number
): number {
  while (left < right) {
    const m1 = left + Math.floor((right - left) / 3)
    const m2 = right - Math.floor((right - left) / 3)

    if (f(m1) < f(m2)) {
      left = m1 + 1    // 极大值在 m1 右侧
    } else {
      right = m2 - 1   // 极大值在 m2 左侧
    }
  }

  return left
}
```

注意整数版本中 `left = m1 + 1` 和 `right = m2 - 1`，因为 m1 和 m2 已经被检查过，可以排除。

### 时间复杂度分析

- **浮点版本**：每次迭代区间变为 2/3，需要 O(log_{1.5}((r-l)/epsilon)) 次迭代
- **整数版本**：需要 O(log_{1.5}(r-l)) 次迭代
- 每次迭代需要 2 次函数求值

实际应用中，浮点版本通常在 50-100 次迭代内收敛。

## 可视化说明

在可视化面板中，三分搜索的过程以函数曲线图的形式直观展示：

- **蓝色曲线**表示单峰函数
- **绿色线段**表示当前搜索区间 [l, r]
- **红色点**表示两个三等分点 m1 和 m2
- **灰色区域**表示已被排除的区间

每一步都会显示：
- 当前区间的左右端点 l、r
- 两个三等分点 m1、m2 的位置和函数值
- 比较结果和排除方向
- 区间长度的变化

通过控制栏，你可以：
- 选择不同的单峰函数
- 逐步执行或自动播放
- 调整播放速度
- 重置到初始状态
- 切换求最大值/最小值模式

## 常见错误

### 1. 将三分搜索应用于非单峰函数

```typescript
// 错误：f(x) = sin(x) 在 [0, 4π] 上有多个极值点
const result = ternarySearch(0, 4 * Math.PI, Math.sin)  // 结果不可靠！

// 正确：确保函数在搜索区间上是单峰的
// 如果函数有多个极值，需要先确定包含目标极值的单峰区间
const result = ternarySearch(0, Math.PI, Math.sin)  // 在 [0, π] 上找最大值
```

### 2. 精度设置不当

```typescript
// 错误：精度太高，导致迭代次数过多
while (right - left > 1e-15) { ... }  // 可能永远不收敛（浮点精度限制）

// 错误：精度太低，结果不精确
while (right - left > 1) { ... }  // 误差可能很大

// 合理的精度设置
while (right - left > 1e-9) { ... }  // 对于大多数应用足够
```

### 3. 整数版本的边界处理

```typescript
// 错误：整数版本使用浮点版本的逻辑
while (left < right) {
  const m1 = left + Math.floor((right - left) / 3)
  const m2 = right - Math.floor((right - left) / 3)
  if (f(m1) < f(m2)) {
    left = m1      // 错误！m1 仍可能是最优解
  } else {
    right = m2     // 错误！m2 仍可能是最优解
  }
}

// 正确：排除已检查的点
if (f(m1) < f(m2)) {
  left = m1 + 1    // m1 不可能是最优解
} else {
  right = m2 - 1   // m2 不可能是最优解
}
```

### 4. 混淆求最大值和最小值的比较方向

```typescript
// 求最大值：f(m1) < f(m2) 时，极值在右侧
if (f(m1) < f(m2)) {
  left = m1
}

// 求最小值：f(m1) < f(m2) 时，极值在左侧（m1 更小，说明极小值在左边）
if (f(m1) < f(m2)) {
  right = m2
}
```

### 5. 忘记检查区间收敛条件

```typescript
// 错误：整数版本使用 <= 导致可能死循环
while (left <= right) { ... }

// 正确：整数版本使用 <
while (left < right) { ... }
```

## 实际应用

### 1. 凸函数优化

对于凸函数（或凹函数），三分搜索可以直接找到全局最优解。例如在机器学习中调整超参数时，如果损失函数关于某个参数是凸的，可以用三分搜索快速找到最优值。

### 2. 计算几何问题

在很多几何问题中，距离函数是单峰的：

- **旋转卡壳**：在凸包上找最远点对
- **最小覆盖圆**：找最优圆心位置
- **最近点对**：在特定约束下的距离最小化

### 3. 游戏理论与博弈

在一些博弈问题中，收益函数关于某个决策变量是单峰的，可以用三分搜索找到最优策略。

### 4. 物理模拟

在物理模拟中，很多优化问题的目标函数是单峰的，例如：

- 找到抛物运动的最高点
- 优化弹簧系统的平衡位置
- 寻找光线折射的最短路径

### 5. 竞赛中的经典应用

三分搜索在算法竞赛中经常与以下技巧结合：

- **答案二分 + 三分验证**：外层二分答案，内层三分验证可行性
- **凸包上的三分**：在凸包的边或顶点上三分搜索
- **函数复合**：多层三分嵌套求解多变量优化

## 总结

三分搜索是一种简单而强大的单峰函数优化算法：

**核心思想**：将搜索区间三等分，通过比较两个分点的函数值，每次排除 1/3 的区间。

**时间复杂度**：O(log_{1.5} n) —— 每次迭代需要两次函数求值。

**空间复杂度**：O(1) —— 只需要几个变量。

**关键要点**：

- 前提条件：函数在搜索区间上必须是单峰的
- 分点计算：m1 = l + (r-l)/3, m2 = r - (r-l)/3
- 比较策略：求最大值时 f(m1) < f(m2) 则 left = m1；求最小值则相反
- 精度控制：浮点版本需要设置合理的 epsilon，避免过度迭代
- 整数版本：使用 m1+1 和 m2-1 来排除已检查的点

掌握三分搜索不仅能解决大量优化问题，还能帮助理解更高级的优化算法（如黄金分割搜索、斐波那契搜索等）的原理。它是连接基础搜索和数值优化的重要桥梁。
