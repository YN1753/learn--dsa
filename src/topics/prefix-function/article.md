# 前缀函数 (Prefix Function / π 函数)

## 概念解释

前缀函数是字符串处理中最基础、最重要的工具之一。给定一个长度为 n 的字符串 s，它的**前缀函数**定义为一个长度为 n 的数组 π，其中 π[i] 表示子串 s[0..i] 的**最长真前缀**同时也是该子串**后缀**的长度。

形式化定义：

```
π[i] = max{ k : k < i+1 且 s[0..k-1] = s[i-k+1..i] }
```

其中 s[0..k-1] 是子串 s[0..i] 的真前缀，s[i-k+1..i] 是子串 s[0..i] 的后缀。

**注意**：这里要求的是"真前缀"，即前缀不能等于整个字符串本身。

### 直观理解

想象一个字符串 "ababaca"：

```
索引:    0  1  2  3  4  5  6
字符:    a  b  a  b  a  c  a
π 数组:  0  0  1  2  3  0  1
```

- π[0] = 0：单个字符没有真前缀
- π[2] = 1：s[0..2] = "aba"，前缀 "a" = 后缀 "a"
- π[3] = 2：s[0..3] = "abab"，前缀 "ab" = 后缀 "ab"
- π[4] = 3：s[0..4] = "ababa"，前缀 "aba" = 后缀 "aba"
- π[5] = 0：s[0..5] = "ababac"，无相等前后缀
- π[6] = 1：s[0..6] = "ababaca"，前缀 "a" = 后缀 "a"

## 为什么重要

前缀函数在字符串算法中具有核心地位：

1. **KMP 算法的基础**：KMP 模式匹配算法的失配函数（failure function）就是前缀函数的变体
2. **线性时间复杂度**：前缀函数可以在 O(n) 时间内计算完成，是许多字符串问题的加速关键
3. **周期性检测**：通过前缀函数可以快速发现字符串的周期性结构
4. **广泛应用**：文本编辑器的查找功能、DNA 序列分析、数据压缩等领域都依赖前缀函数

## 核心原理

### 朴素算法 O(n³)

最直观的方法：对每个位置 i，从最长的可能长度开始，逐一检查前缀是否等于后缀。

```typescript
function prefixFunctionNaive(s: string): number[] {
  const n = s.length
  const pi: number[] = new Array(n).fill(0)

  for (let i = 0; i < n; i++) {
    for (let k = 0; k < i; k++) {
      // 检查 s[0..k] 是否等于 s[i-k..i]
      let match = true
      for (let j = 0; j <= k; j++) {
        if (s[j] !== s[i - k + j]) {
          match = false
          break
        }
      }
      if (match) {
        pi[i] = k + 1
      }
    }
  }
  return pi
}
```

**时间复杂度**：O(n³) — 三层嵌套循环。效率太低，不实用。

### 优化算法 O(n²)

改进：对每个位置 i，先假设 π[i] = π[i-1] + 1，然后逐步缩短直到找到匹配。

```typescript
function prefixFunctionOptimized(s: string): number[] {
  const n = s.length
  const pi: number[] = new Array(n).fill(0)

  for (let i = 1; i < n; i++) {
    let j = pi[i - 1]
    while (j > 0 && s[i] !== s[j]) {
      j = pi[j - 1]  // 利用已计算的值跳转
    }
    if (s[i] === s[j]) {
      j++
    }
    pi[i] = j
  }
  return pi
}
```

等等，这个代码实际上已经是 O(n) 了！让我解释为什么。

### 高效算法 O(n) — 核心思想

高效算法的关键洞察：

**当我们计算 π[i] 时，已经知道了 π[0], π[1], ..., π[i-1] 的值。这些信息可以帮助我们快速跳转，而不是从头开始比较。**

算法流程：

1. 维护一个变量 j，表示当前已匹配的前缀长度
2. 对于位置 i，如果 s[i] = s[j]，则 j++，π[i] = j
3. 如果 s[i] ≠ s[j]，则利用 π[j-1] 的值跳转到更短的候选前缀
4. 重复步骤 2-3 直到匹配或 j = 0

```
示例：s = "aabaaab"

i=0: π[0] = 0 (单个字符)
i=1: s[1]='a', s[0]='a', 匹配! j=1, π[1]=1
i=2: s[2]='b', s[1]='a', 不匹配, j=π[0]=0, s[2]='b', s[0]='a', 不匹配, π[2]=0
i=3: s[3]='a', s[0]='a', 匹配! j=1, π[3]=1
i=4: s[4]='a', s[1]='a', 匹配! j=2, π[4]=2
i=5: s[5]='a', s[2]='b', 不匹配, j=π[1]=1, s[5]='a', s[1]='a', 匹配! j=2, π[5]=2
i=6: s[6]='b', s[2]='b', 匹配! j=3, π[6]=3
```

**为什么是 O(n)？**

通过摊还分析：变量 j 在整个算法中最多增加 n 次（每次匹配成功 j++），也最多减少 n 次（每次跳转 j 减小）。因此 while 循环的总执行次数为 O(n)，每个字符只被处理常数次。

### 与 KMP 失败函数的关系

KMP 算法中的失败函数（也叫 next 数组）本质上就是前缀函数的变体：

| 概念 | 前缀函数 π | KMP next 数组 |
|------|-----------|---------------|
| 定义 | s[0..i] 的最长相等前后缀长度 | 模式串在位置 j 失配时的回退位置 |
| 关系 | π[i] | next[j+1] = π[j] |
| 索引 | 从 0 开始 | 通常从 1 开始 |

简单来说：**next[j] = π[j-1]**（忽略边界处理的差异）。

## 可视化说明

在可视化界面中，前缀函数的计算过程展示了：

- **字符串的字符排列**：每个字符用方框表示，清晰显示索引位置
- **前后缀匹配**：高亮显示当前正在比较的前缀和后缀部分
- **π 数组的变化**：实时显示每个位置的 π 值
- **跳转过程**：当不匹配时，展示如何利用 π 值进行快速跳转

通过动画可以直观理解：
- 为什么朴素算法效率低（大量重复比较）
- 高效算法如何利用已有信息跳过不必要的比较
- π 值如何反映字符串的内部结构

## 常见错误

### 1. 混淆真前缀和前缀

```typescript
// 错误：允许整个字符串作为前缀
// s = "aaa", 错误地认为 π[2] = 3
// 正确：真前缀不能等于字符串本身，π[2] = 2

// 真前缀 (proper prefix): s[0..k] where k < n-1
// 前缀 (prefix): s[0..k] where k <= n-1
```

### 2. 忘记处理边界情况

```typescript
// 错误：从 i=0 开始循环
for (let i = 0; i < n; i++) { ... }

// 正确：π[0] 恒为 0，从 i=1 开始
pi[0] = 0
for (let i = 1; i < n; i++) { ... }
```

### 3. 跳转时的索引错误

```typescript
// 错误：跳转时使用 pi[i-1] 而不是 pi[j-1]
j = pi[i - 1]  // 错误！

// 正确：应该使用 pi[j-1]
j = pi[j - 1]  // 正确
```

### 4. 误解 π 值的含义

```
常见误解：π[i] 表示以位置 i 结尾的最长回文子串长度
正确含义：π[i] 表示 s[0..i] 的最长相等真前后缀长度

例如 s = "ababa"
π[4] = 3，表示 "ababa" 的前缀 "aba" 等于后缀 "aba"
但这不是回文，只是前后缀恰好相同
```

## 实际应用

### 1. KMP 模式匹配

前缀函数最经典的应用是在 O(n+m) 时间内完成字符串模式匹配：

```typescript
function kmpSearch(text: string, pattern: string): number[] {
  const combined = pattern + '#' + text
  const pi = computePrefixFunction(combined)
  const m = pattern.length
  const matches: number[] = []

  for (let i = m + 1; i < combined.length; i++) {
    if (pi[i] === m) {
      matches.push(i - 2 * m)
    }
  }
  return matches
}
```

### 2. 字串周期检测

利用前缀函数可以快速判断字符串是否具有周期性：

```typescript
function findPeriod(s: string): number {
  const pi = computePrefixFunction(s)
  const n = s.length
  const period = n - pi[n - 1]

  if (n % period === 0) {
    return period  // s 由长度为 period 的子串重复构成
  }
  return n  // 无周期
}

// "abcabcabc" -> period = 3 (由 "abc" 重复 3 次)
// "abcdef" -> period = 6 (无周期)
```

### 3. 统计不同子串数量

前缀函数可以帮助高效计算字符串中不同子串的数量，这在生物信息学中有重要应用。

### 4. 字符串压缩

利用周期性检测，可以找到字符串的最小周期，实现最优压缩：

```
"abababab" -> 周期 = 2, 压缩为 "ab" x 4
"abcabcabc" -> 周期 = 3, 压缩为 "abc" x 3
```

## 总结

前缀函数是字符串算法的基石：

**核心思想**：
- π[i] = 子串 s[0..i] 的最长相等真前后缀长度
- 利用已计算的 π 值加速后续计算，实现 O(n) 时间复杂度

**关键特性**：
- 朴素算法 O(n³)，优化后可达 O(n)
- 与 KMP 失配函数本质相同
- 可用于检测字符串的周期性结构

**典型应用**：
- KMP 字符串匹配 O(n+m)
- 字符串周期性检测
- 字符串压缩
- 生物信息学中的序列分析

掌握前缀函数是理解高级字符串算法（如 Z 函数、后缀数组、Aho-Corasick 自动机）的重要前提。
