# 欧拉路径 (Eulerian Path)

## 概念解释

**欧拉路径**是指经过图中**每条边恰好一次**的路径。如果这条路径的起点和终点相同（即形成一个回路），则称为**欧拉回路**（Eulerian Circuit）。

这两个概念以数学家莱昂哈德·欧拉（Leonhard Euler）命名。1736年，欧拉在研究柯尼斯堡七桥问题时，首次提出了图论的基本思想，开创了离散数学的全新领域。

**关键区别：**
- **欧拉路径/回路**：经过每条**边**恰好一次
- **哈密顿路径/回路**：经过每个**顶点**恰好一次

这是两个完全不同层次的问题。欧拉路径有多项式时间算法（O(E)），而哈密顿路径是NP完全问题。

## 为什么重要

### 1. 图论的起源——柯尼斯堡七桥问题

18世纪的柯尼斯堡（今俄罗斯加里宁格勒）有七座桥连接城市的两个岛屿和两岸。市民们想知道：能否一次走遍所有七座桥，且每座桥只走一次？欧拉将陆地抽象为顶点、桥抽象为边，证明了这样的路径不存在。这一工作标志着**图论**学科的诞生。

### 2. DNA片段组装

在生物信息学中，DNA测序会产生大量短片段（reads）。科学家需要将这些片段拼接成完整的基因组序列。通过将片段重叠关系建模为图，寻找欧拉路径可以高效地完成DNA组装任务。这是现代基因组学中**de Bruijn图**方法的核心。

### 3. 电路板设计与布线

在印刷电路板（PCB）设计中，需要规划一条路径遍历所有导线连接点而不重复。欧拉路径理论可以帮助设计最优的布线方案，减少制造成本和时间。

### 4. 网络路由优化

在计算机网络中，某些巡检或遍历任务需要经过每条链路恰好一次。欧拉路径提供了理论基础，帮助设计高效的路由算法。

## 核心原理

### 存在性条件

判断一个图是否存在欧拉路径/回路，是解决问题的第一步。

#### 无向图

| 条件 | 结论 |
|------|------|
| 连通 + 所有顶点度数为偶数 | 存在**欧拉回路** |
| 连通 + 恰好2个奇数度顶点 | 存在**欧拉路径**（从一个奇数度顶点到另一个） |
| 连通 + 奇数度顶点数为0 | 存在**欧拉回路**（也是欧拉路径的特例） |
| 不连通 或 奇数度顶点数 > 2 | **不存在**欧拉路径 |

**直觉理解：** 每次经过一个顶点，都需要消耗两条边（一条进入，一条离开）。因此，除了起点和终点外，所有中间顶点的度数必须是偶数。

#### 有向图

| 条件 | 结论 |
|------|------|
| 弱连通 + 所有顶点入度 = 出度 | 存在**欧拉回路** |
| 弱连通 + 恰好1个顶点出度 = 入度+1，恰好1个顶点入度 = 出度+1，其余相等 | 存在**欧拉路径** |

### Hierholzer 算法

Hierholzer算法是求欧拉路径/回路的最优算法，核心思想是**DFS + 回路拼接**。

**算法步骤：**

1. **选择起点**：欧拉回路任选，欧拉路径选奇数度顶点
2. **贪心行走**：从当前顶点出发，选择一条未访问的边，走到下一个顶点，并删除已走过的边
3. **回溯拼接**：当走到无路可走时（当前顶点无未访问的边），将该顶点加入路径，回溯到上一个顶点
4. **重复**：继续步骤2-3，直到所有边都被访问
5. **得到路径**：最终路径的逆序就是欧拉路径/回路

```typescript
function hierholzer(adj: Map<number, number[]>, start: number): number[] {
  const path: number[] = []
  const stack: number[] = [start]

  while (stack.length > 0) {
    const v = stack[stack.length - 1]
    if (adj.get(v)!.length === 0) {
      // 无路可走，加入路径
      path.push(stack.pop()!)
    } else {
      // 贪心选择一条边
      const u = adj.get(v)!.pop()!
      // 移除反向边
      const rev = adj.get(u)!
      rev.splice(rev.indexOf(v), 1)
      stack.push(u)
    }
  }

  return path.reverse()
}
```

**为什么有效？** Hierholzer算法的本质是在图中不断寻找简单回路，然后将它们拼接在一起。每次DFS走到死胡同时，说明当前路径形成了一个回路（或路径的终点）。通过回溯，算法会在还有未使用边的顶点处"插入"新的子回路，最终拼接成完整的欧拉路径。

### Fleury 算法（对比）

Fleury算法是另一种求欧拉路径的算法，但效率较低：

1. 从起点出发，每次选择一条未访问的边
2. **关键约束**：如果有桥边和非桥边可选，优先选非桥边（避免过早断开图）
3. 删除已走过的边，继续行走

**缺点**：每次选择边时需要判断是否为桥，时间复杂度为O(E²)。

## 常见错误

### 1. 混淆欧拉路径与哈密顿路径

这是最常见的错误。记住：
- **欧拉**：关注**边**，经过每条边恰好一次
- **哈密顿**：关注**顶点**，经过每个顶点恰好一次

### 2. 忘记检查连通性

即使度数条件满足，如果图不连通，也不存在欧拉路径。例如：两个分离的三角形，每个顶点度数都是2（偶数），但整体不连通，没有欧拉回路。

### 3. 错误判断有向图的条件

有向图的条件是关于**入度和出度**的关系，而不是简单的度数奇偶性。初学者经常将无向图的条件错误地套用到有向图上。

### 4. Hierholzer实现中忘记删除反向边

在无向图中，每条边(u,v)在邻接表中出现两次。删除边时必须同时删除两个方向，否则会导致边被重复访问。

### 5. 路径方向搞混

Hierholzer算法生成的路径是**逆序**的（从回溯过程收集），需要反转后才是正确的欧拉路径方向。

## 实际应用

### 1. 柯尼斯堡七桥问题（历史起源）

这是欧拉路径理论的起源。欧拉证明了：当且仅当图中奇数度顶点数为0或2时，才存在欧拉路径/回路。柯尼斯堡的桥图有4个奇数度顶点，因此无解。

### 2. DNA片段组装（de Bruijn图）

现代基因组测序将DNA打碎成短片段，通过构建de Bruijn图（k-mer为节点，(k+1)-mer重叠为边），在图中寻找欧拉路径即可重建原始序列。SPAdes、Velvet等主流基因组组装工具都基于此原理。

### 3. 中国邮递员问题

邮递员需要走遍所有街道至少一次再回到邮局。如果图有欧拉回路，直接走即可；否则需要重复某些边使所有顶点度数变为偶数，目标是使重复的总距离最小。

### 4. 电路板测试

在PCB制造中，需要测试所有导线连接。通过寻找欧拉路径，可以设计一条测试路径，用最少的时间遍历所有连接。

### 5. 一笔画问题

判断一个图形能否"一笔画"（笔不离纸、线不重复）就是判断对应图是否存在欧拉路径。这是欧拉路径理论最直观的日常应用。

## 总结

欧拉路径是图论中最经典的问题之一，具有深远的理论意义和广泛的实际应用：

- **核心思想**：经过每条边恰好一次
- **存在条件**：连通 + 0或2个奇数度顶点（无向图）
- **最优算法**：Hierholzer算法，O(E)时间复杂度
- **关键技巧**：DFS + 回路拼接，贪心选择边并回溯
- **历史意义**：柯尼斯堡七桥问题，开创了图论学科
- **现代应用**：DNA组装、电路设计、网络路由、邮递员问题

理解欧拉路径不仅帮助你解决图论问题，更重要的是体会**抽象建模**的力量——将现实问题转化为图模型，利用数学理论高效求解。
