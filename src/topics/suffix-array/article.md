# 后缀数组 (Suffix Array)

## 概念解释

后缀数组是一种强大的字符串数据结构，它将一个字符串的所有后缀按字典序排列，从而实现高效的字符串查询操作。

### 什么是后缀

对于一个长度为 n 的字符串 S，它有 n 个后缀。后缀 Suffix(i) 是从位置 i 开始到字符串末尾的子串。

```
字符串: "banana"  (长度 n = 6)

Suffix(0) = "banana"
Suffix(1) = "anana"
Suffix(2) = "nana"
Suffix(3) = "ana"
Suffix(4) = "na"
Suffix(5) = "a"
```

### 后缀数组 SA

后缀数组（Suffix Array）是一个整数数组 SA，其中 SA[i] 表示字典序排名第 i 的后缀的起始位置。

```
字符串: "banana"

所有后缀按字典序排序:
  排名  起始位置  后缀
  0     5         "a"
  1     3         "ana"
  2     1         "anana"
  3     0         "banana"
  4     4         "na"
  5     2         "nana"

后缀数组 SA = [5, 3, 1, 0, 4, 2]
```

### Rank 数组

Rank 数组是 SA 的逆数组，Rank[i] 表示以位置 i 开始的后缀在排序后的排名。

```
SA  = [5, 3, 1, 0, 4, 2]
Rank = [3, 2, 5, 1, 4, 0]

解释: Rank[0] = 3，表示 Suffix(0) = "banana" 排名第 3
      Rank[5] = 0，表示 Suffix(5) = "a" 排名第 0（最小）
```

### LCP 数组

LCP（Longest Common Prefix，最长公共前缀）数组记录排序后相邻两个后缀的最长公共前缀长度。

```
LCP[i] = Suffix(SA[i]) 和 Suffix(SA[i-1]) 的最长公共前缀长度
LCP[0] 无定义（通常设为 0）

SA[0]=5 "a"        LCP[0] = 0
SA[1]=3 "ana"      LCP[1] = 1  (与 "a" 的公共前缀 "a")
SA[2]=1 "anana"    LCP[2] = 3  (与 "ana" 的公共前缀 "ana")
SA[3]=0 "banana"   LCP[3] = 0  (与 "anana" 无公共前缀)
SA[4]=4 "na"       LCP[4] = 0  (与 "banana" 无公共前缀)
SA[5]=2 "nana"     LCP[5] = 2  (与 "na" 的公共前缀 "na")

LCP = [0, 1, 3, 0, 0, 2]
```

## 为什么重要

### 1. 高效的子串搜索

在后缀数组上使用二分查找，可以在 O(m log n) 时间内完成模式匹配（m 为模式串长度，n 为文本长度），无需预处理文本。

### 2. 最长公共子串

利用 LCP 数组，可以在 O(n) 时间内求出两个字符串的最长公共子串，这在生物信息学中用于 DNA 序列比对。

### 3. 最长重复子串

LCP 数组中的最大值即为字符串中最长重复子串的长度，这在数据压缩和文本分析中非常有用。

### 4. 空间效率

后缀数组仅需要 O(n) 的额外空间（不包括 LCP 数组），相比后缀树的指针开销更加节省内存。

## 核心原理

### 朴素构建方法

最简单的方法是生成所有后缀，然后排序。

```
构建后缀数组:
1. 生成所有后缀 (起始位置 0, 1, ..., n-1)
2. 对后缀进行字典序排序
3. SA[i] = 排名第 i 的后缀的起始位置

时间复杂度: O(n² log n)  (n 个后缀，每次比较 O(n))
空间复杂度: O(n)
```

### 倍增算法 (Doubling Algorithm)

倍增算法是经典的 O(n log²n) 构建方法。核心思想是：从长度 1 开始，每次将排序关键字的长度翻倍。

```
倍增算法步骤:
1. 初始: 对所有单字符排序 (长度 k=1)
2. 迭代: 利用长度 k 的排序结果，对长度 2k 排序
3. 重复: k 翻倍，直到 2k >= n

每轮排序使用关键字 (Rank[i], Rank[i+k]) 的双关键字排序
使用基数排序可以在 O(n) 时间内完成每轮排序
总时间: O(n log n) 轮次 × O(n) 每轮 = O(n log n)
```

### Kasai 算法构建 LCP 数组

Kasai 算法可以在 O(n) 时间内从后缀数组构建 LCP 数组。

```
Kasai 算法:
1. 计算 Rank 数组
2. 遍历原字符串的每个位置 i (按 i 的顺序，不是 SA 的顺序)
3. 对于位置 i，找到它在 SA 中的排名 r = Rank[i]
4. 比较 Suffix(SA[r]) 和 Suffix(SA[r-1]) 的公共前缀
5. 利用 h 变量避免重复比较: h = max(0, h-1)

关键观察: Suffix(i) 和 Suffix(j) 的 LCP 减 1
          等于 Suffix(i+1) 和 Suffix(j+1) 的 LCP 的下界
```

### 基于后缀数组的模式匹配

使用二分查找在后缀数组中定位模式串。

```
模式匹配算法:
1. 在 SA 上二分查找，找到第一个 >= pattern 的后缀
2. 再找到第一个 > pattern 的后缀（即 pattern 加上一个大字符）
3. 两个位置之间的 SA 条目就是所有匹配位置

时间: O(m log n)，m 为模式串长度
```

在可视化界面中，你可以：

- **观察后缀排序过程**：看后缀如何按字典序排列
- **高亮 LCP 区域**：相邻后缀的公共前缀用颜色标记
- **逐步演示二分查找**：输入模式串，观察搜索范围如何缩小
- **调整播放速度**：通过速度滑块控制动画节奏

## 常见错误

### 1. 混淆后缀数组和后缀树

后缀数组是数组结构，后缀数组 + LCP 数组可以模拟后缀树的大部分功能，但它们是不同的数据结构。后缀树是树形结构，支持在线构建。

### 2. LCP 数组的下标偏移

LCP[i] 对应的是 SA[i] 和 SA[i-1] 的公共前缀长度，不是 SA[i] 和 SA[i+1]。LCP[0] 通常没有意义，设为 0。

### 3. 未正确处理哨兵字符

在构建后缀数组时，通常需要在字符串末尾添加一个哨兵字符（如 '$' 或 '\0'），且哨兵字符必须小于所有其他字符。这确保了所有后缀的长度不同，避免排序时的歧义。

### 4. 倍增算法中的双关键字排序

排序时需要注意：第二关键字为 0 的情况（即 i+k >= n 时），应排在最前面。

### 5. Rank 和 SA 的关系

Rank[SA[i]] = i，SA[Rank[i]] = i。混淆这两者是常见错误。

## 实际应用

### 1. 生物信息学

在 DNA 序列分析中，后缀数组用于快速搜索基因模式。人类基因组约 30 亿碱基对，后缀数组可以在毫秒级内找到特定序列的所有出现位置。

### 2. 数据压缩

Burrows-Wheeler 变换（BWT）是许多压缩算法（如 bzip2）的核心，BWT 可以直接从后缀数组计算得出。BWT 将相似的字符聚集在一起，提高压缩率。

### 3. 抄袭检测

通过比较文档的后缀数组和 LCP 数组，可以高效地检测文本之间的相似度和重复片段。

### 4. 全文搜索引擎

搜索引擎使用后缀数组（或其变体）来实现快速的全文检索。配合 LCP 数组，可以支持通配符搜索和模糊匹配。

### 5. 最长公共子串问题

给定多个字符串，可以通过合并字符串并构建后缀数组 + LCP 数组，在 O(n log n) 时间内找到最长公共子串。

## 总结

后缀数组是一种简洁而强大的字符串数据结构。它通过将所有后缀排序，将复杂的字符串问题转化为数组上的二分查找和区间查询问题。

**核心优势：**
- 实现简单，代码量远少于后缀树
- 空间效率高，仅需 O(n) 额外空间
- 配合 LCP 数组功能强大，可解决大多数字符串查询问题
- 常数因子小，实际运行速度快

**适用场景：**
- 需要在大文本中多次搜索不同模式串
- 需要找最长公共子串或最长重复子串
- 需要高效的字符串索引结构
- 内存受限但需要处理大规模文本

掌握后缀数组，你就拥有了处理复杂字符串问题的利器。
