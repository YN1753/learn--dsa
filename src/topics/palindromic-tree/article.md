# 回文树（Eertree / Palindromic Tree）

## 概念解释

回文树（Palindromic Tree），也称为 **Eertree**，是一种用于处理回文子串问题的高效数据结构。它能够在线性时间内构建，并且能够：

- 统计字符串中**本质不同的回文子串**数量
- 快速查询每个位置的**最长回文后缀**
- 统计每个回文子串的**出现次数**

### 核心思想

回文树将一个字符串中所有本质不同的回文子串组织成一棵树形结构。与普通Trie树类似，每个节点代表一个回文子串，节点之间的转移边代表在当前回文串两端添加一个字符。

### 两棵Trie

回文树的独特之处在于它包含**两棵树**：

- **奇根（odd root）**：长度为 -1 的虚拟节点，用于处理所有**奇数长度**的回文子串
- **偶根（even root）**：长度为 0 的虚拟节点，用于处理所有**偶数长度**的回文子串

这种双根设计巧妙地避免了在插入过程中的边界处理问题。

## 为什么重要

回文子串问题是字符串算法中的经典问题。传统方法需要 O(n^2) 甚至 O(n^3) 的时间复杂度，而回文树提供了以下优势：

1. **线性时间构建**：O(n) 时间内处理整个字符串
2. **本质不同回文子串计数**：直接通过节点数得到
3. **在线算法**：逐字符处理，支持动态扩展
4. **空间高效**：节点数不超过 n+2（n 为字符串长度）
5. **强大的查询能力**：支持回文子串出现次数、最长回文后缀等查询

### 与其他算法的对比

| 算法 | 时间复杂度 | 空间复杂度 | 特点 |
|------|------------|------------|------|
| 暴力枚举 | O(n^3) | O(1) | 简单但低效 |
| Manacher | O(n) | O(n) | 找所有回文子串位置 |
| 回文树 | O(n) | O(n) | 本质不同回文子串，支持计数 |

## 核心原理

### 节点结构

每个节点包含以下信息：

```typescript
interface EertreeNode {
  len: number      // 该回文子串的长度
  link: number     // 后缀链接，指向最长回文真后缀
  next: Map<string, number>  // 转移边：字符 -> 子节点编号
  count: number    // 该回文子串的出现次数
}
```

### 后缀链接（Suffix Link）

后缀链接是回文树的核心概念。对于每个节点 u（代表回文串 P），其后缀链接指向节点 v，其中 v 代表 P 的**最长回文真后缀**。

例如，对于回文串 "ababa"：
- 它的最长回文真后缀是 "aba"
- 因此 "ababa" 节点的后缀链接指向 "aba" 节点

后缀链接的作用类似于AC自动机中的 fail 指针，使得在当前回文串无法扩展时，能快速跳转到下一个候选回文串。

### 构建过程

逐字符处理字符串，对于每个新字符 s[i]：

1. **寻找可扩展节点**：从当前最长回文后缀节点开始，沿后缀链接向上跳转，直到找到一个节点 u，使得 s[i - len(u) - 1] == s[i]

2. **检查转移是否存在**：如果节点 u 通过字符 s[i] 已经有转移边，说明该回文子串已存在，直接更新

3. **创建新节点**：如果没有转移边，创建新节点，长度为 len(u) + 2

4. **设置后缀链接**：对于新节点，继续沿后缀链接寻找其最长回文真后缀

```typescript
// 伪代码
for each character s[i]:
  cur = last  // last 指向当前最长回文后缀
  while s[i - len(cur) - 1] != s[i]:
    cur = link[cur]  // 沿后缀链接跳转

  if s[i] in next[cur]:
    last = next[cur][s[i]]  // 已存在
  else:
    create new node with len = len(cur) + 2
    find suffix link for new node
    last = new node
```

### 时间复杂度分析

虽然看起来有嵌套循环，但通过均摊分析可以证明总时间复杂度为 O(n)：

- 每次沿后缀链接跳转，当前回文后缀长度严格减小
- 对于每个字符，跳转次数的均摊值为 O(1)
- 因此总体为 O(n)

## 可视化说明

在可视化界面中，回文树展示为：

- **椭圆节点**：每个节点代表一个本质不同的回文子串，标注其长度
- **实线箭头**：字符转移边，表示在回文串两端添加该字符
- **虚线箭头**：后缀链接，指向最长回文真后缀节点
- **双根结构**：奇根（len=-1）和偶根（len=0）位于顶部

4. 当前活跃节点用蓝色高亮

## 常见错误

### 1. 忘记初始化两个根节点

```typescript
// 错误：只创建了一个根
const nodes = [{ len: 0, link: 0, next: new Map() }]

// 正确：创建奇根和偶根
const nodes = [
  { len: -1, link: 0, next: new Map() },  // 奇根
  { len: 0, link: 0, next: new Map() },   // 偶根
]
```

### 2. 后缀链接设置错误

```typescript
// 错误：新节点的后缀链接直接指向偶根
newNode.link = 1

// 正确：需要沿后缀链接寻找真正的最长回文真后缀
let linkCur = nodes[cur].link
while (s[i - nodes[linkCur].len - 1] !== s[i]) {
  linkCur = nodes[linkCur].link
}
newNode.link = nodes[linkCur].next.get(s[i])
```

### 3. 边界条件：长度为1的回文

长度为 1 的回文子串的后缀链接应该直接指向偶根，而不是继续寻找。

```typescript
if (newNode.len === 1) {
  newNode.link = 1  // 指向偶根
  return
}
```

### 4. 混淆「本质不同」和「出现次数」

回文树的节点数等于**本质不同**回文子串的数量，而不是总出现次数。如果需要统计出现次数，需要额外维护 count 字段并在构建完成后进行传播。

## 实际应用

### 1. 回文子串计数

最直接的应用：统计字符串中有多少个本质不同的回文子串。

```typescript
const distinctPalindromes = nodeCount - 2  // 减去两个根节点
```

### 2. 最长回文子串

通过记录每个位置的最长回文后缀长度，可以快速找到整个字符串的最长回文子串。

### 3. 回文串出现次数

通过后缀链接传播计数，可以得到每个回文子串在字符串中的出现次数。

### 4. 回文分割问题

结合动态规划，回文树可以高效解决回文分割相关的优化问题。

### 5. 字符串匹配

在某些字符串匹配问题中，回文树可以帮助快速判断子串是否为回文。

## 总结

回文树是一种精巧的数据结构，它将回文子串问题从 O(n^2) 降低到 O(n)：

**核心要点**：
- 两棵Trie（奇根和偶根）处理奇偶长度回文
- 后缀链接实现快速跳转，类似AC自动机的 fail 指针
- 在线算法，逐字符构建
- 节点数 = 本质不同回文子串数

**适用场景**：
- 需要统计本质不同回文子串
- 需要知道每个回文子串的出现次数
- 需要在线处理回文相关的查询
- 竞赛中的回文子串问题

**学习建议**：
理解回文树需要先掌握回文的基本性质、后缀链接的概念，以及AC自动机中 fail 指针的思想。建议结合可视化工具观察每一步的构建过程，理解后缀链接的跳转逻辑。
