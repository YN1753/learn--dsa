# KMP自动机 (KMP Automaton)

## 概念解释

KMP自动机是将KMP算法的核心思想转化为**确定有限状态自动机(DFA)**的形式。它把字符串匹配过程建模为一个状态机：读入文本的每个字符后，自动机根据当前状态和输入字符确定地转移到下一个状态。当自动机到达「接受状态」时，意味着找到了一个完整匹配。

### 核心术语

| 术语 | 说明 |
|------|------|
| 状态 (State) | 表示当前已匹配模式串前缀的长度，范围为 0 到 m |
| 转移函数 (Transition) | δ(state, char) → nextState，给定当前状态和输入字符，返回下一状态 |
| 接受状态 (Accept State) | 状态 m（模式串长度），表示完全匹配 |
| 失配函数 (Fail Function) | 与KMP的next数组相同，决定匹配失败时的回退位置 |

### 与普通KMP的区别

普通KMP算法在匹配过程中需要**动态计算**失配函数的回退：

```
普通KMP：匹配失败 → 计算fail → 回退 → 继续比较（可能多次回退）
KMP自动机：匹配失败 → 直接查表转移（一步到位）
```

KMP自动机将所有可能的转移**预先计算**为一张完整的转移表，匹配时无需任何条件判断。

## 为什么重要

KMP自动机在理论和实践上都有重要意义：

1. **理论桥梁**：连接了算法设计（KMP）与形式语言理论（自动机），展示了同一问题的不同建模方式
2. **无回溯匹配**：DFA保证每个输入字符只被读取一次，匹配过程严格O(n)
3. **硬件友好**：转移表可以用纯查表实现，非常适合FPGA/ASIC等硬件加速
4. **多模式扩展**：KMP自动机是理解Aho-Corasick自动机的基础，后者是多模式匹配的标准算法
5. **编译器应用**：词法分析器中的正则表达式引擎底层就是类似的自动机

## 核心原理

### 从fail函数到转移表

KMP自动机的核心是将fail函数（next数组）展开为完整的转移表`dfa[c][j]`，其中`c`是字符，`j`是状态。

**直觉理解**：

假设模式串为 `P = "ABABC"`，当我们在状态4（已匹配"ABAB"）时遇到字符`c`：
- 如果`c == 'C'`，成功匹配，转移到状态5
- 如果`c == 'A'`，直接匹配失败，但利用fail函数回退：fail[4]=2（"ABAB"的最长相等前后缀长度为2），然后检查P[2]='A'与c匹配，所以转移到状态3
- 如果`c == 'B'`，fail[4]=2，检查P[2]='A'与c不匹配，继续回退到fail[2]=0，检查P[0]='A'与c不匹配，停留在状态0

自动机将这些复杂的回退逻辑**一次性预计算**为转移表。

### 预处理算法

```typescript
function buildDFA(pattern: string): number[][] {
  const m = pattern.length
  const R = 256  // 字符集大小（ASCII）
  const dfa: number[][] = Array.from({ length: R }, () => new Array(m).fill(0))

  // 状态0遇到pattern[0]转移到状态1
  dfa[pattern.charCodeAt(0)][0] = 1

  // 用变量x记录"重启状态"
  let x = 0

  for (let j = 1; j < m; j++) {
    // 复制失败转移的列
    for (let c = 0; c < R; c++) {
      dfa[c][j] = dfa[c][x]
    }
    // 设置匹配转移
    dfa[pattern.charCodeAt(j)][j] = j + 1
    // 更新重启状态
    x = dfa[pattern.charCodeAt(x)][x]  // 等价于 fail[j]
  }

  return dfa
}
```

**关键洞察**：第j列的转移表等于`dfa[·][fail[j]]`加上一个匹配转移。这利用了fail函数的递推性质。

### 匹配过程

```typescript
function search(text: string, dfa: number[][], m: number): number[] {
  const results: number[] = []
  let state = 0

  for (let i = 0; i < text.length; i++) {
    state = dfa[text.charCodeAt(i)][state]
    if (state === m) {
      results.push(i - m + 1)  // 找到匹配
      state = 0  // 重置（或继续搜索可设为 dfa[...][m-1]）
    }
  }

  return results
}
```

匹配过程极其简洁：读入字符，查表，更新状态。没有任何循环或条件分支。

### 时间复杂度

| 阶段 | 时间复杂度 | 空间复杂度 |
|------|------------|------------|
| 预处理（构建DFA） | O(m × \|Σ\|) | O(m × \|Σ\|) |
| 匹配 | O(n) | O(1) |

其中 m 是模式串长度，n 是文本长度，\|Σ\| 是字符集大小。

### 与朴素算法对比

| 特性 | 朴素算法 | KMP算法 | KMP自动机 |
|------|----------|---------|-----------|
| 预处理 | 无 | O(m) | O(m × \|Σ\|) |
| 最坏匹配 | O(nm) | O(n) | O(n) |
| 回溯 | 需要 | 不需要 | 不需要 |
| 匹配中条件分支 | 多 | 少 | 几乎无 |
| 硬件实现 | 困难 | 一般 | 非常适合 |

## 可视化说明

在可视化界面中，KMP自动机展示为一个**有向图**：

- **节点**表示状态（0到m），标注当前已匹配的前缀
- **有向边**表示转移，标注触发转移的字符
- **粗边**表示成功匹配转移（状态+1）
- **虚边**表示失配回退转移
- **高亮节点**表示当前自动机所处的状态

通过可视化可以观察：
- 自动机如何在状态之间跳转
- 失配时如何利用fail函数优雅地回退
- 匹配过程中每个字符只被读取一次

## 常见错误

### 1. 混淆转移表的行列含义

```typescript
// 错误：把 dfa[j][c] 当作"状态j遇到字符c"
const next = dfa[state][charCode]  // 反了！

// 正确：dfa[charCode][state]
const next = dfa[charCode][state]
```

转移表的常见布局是`dfa[字符][状态]`，按列存储更缓存友好（因为匹配时状态变化频繁，字符随机）。

### 2. 忘记处理接受状态的后续转移

```typescript
// 错误：匹配后直接重置为0
if (state === m) {
  results.push(i - m + 1)
  state = 0  // 可能错过重叠匹配！
}

// 正确：如果需要找所有匹配（包括重叠）
if (state === m) {
  results.push(i - m + 1)
  state = dfa[text.charCodeAt(i)][m - 1]  // 从fail[m]继续
}
```

### 3. 字符集大小选择不当

```typescript
// 问题：只考虑小写字母
const R = 26
dfa[pattern.charCodeAt(j) - 'a'.charCodeAt(0)][j] = ...  // 遇到大写或数字就崩溃

// 安全：使用完整ASCII或Unicode
const R = 256
dfa[pattern.charCodeAt(j)][j] = ...
```

### 4. 误以为KMP自动机总是优于普通KMP

KMP自动机的空间复杂度为O(m × |Σ|)。当字符集很大（如Unicode）时，转移表可能占用大量内存。此时普通KMP（空间O(m)）或使用稀疏转移表（HashMap）更合适。

## 实际应用

### 1. 文本编辑器的查找功能

文本编辑器需要在文档中快速查找关键词。KMP自动机的O(n)匹配和无回溯特性使其非常适合流式处理大文件。

### 2. 生物信息学中的基因序列匹配

DNA序列只有4种碱基(A, T, G, C)，字符集很小，KMP自动机的转移表非常紧凑，适合在海量基因数据中高效匹配特定序列片段。

### 3. 网络入侵检测系统

IDS/IPS需要在实时网络流量中检测恶意签名。KMP自动机可以作为硬件加速器的算法基础，实现线速匹配。

### 4. 编译器词法分析

编译器的词法分析器将源代码分解为token。虽然现代编译器使用更复杂的工具（如flex），但其底层原理与KMP自动机相同：将正则表达式编译为DFA，然后用DFA扫描输入。

### 5. Aho-Corasick自动机的基础

AC自动机在Trie树上构建KMP风格的fail指针，实现多模式同时匹配。理解KMP自动机是学习AC自动机的必要前提。

## 总结

KMP自动机是将KMP算法的形式化、自动机化版本：

**核心思想**：
- 将模式串编码为一个确定有限状态自动机(DFA)
- 预计算所有可能的转移（包括成功和失败情况）
- 匹配过程简化为纯查表操作

**关键特性**：
- 预处理 O(m × |Σ|)，匹配 O(n)
- 无回溯、无条件分支的匹配过程
- 理论优雅，实践高效

**适用场景**：
- 字符集较小且模式串固定的场景
- 需要硬件加速的高速匹配
- 理解自动机理论的入门案例
- 作为AC自动机等高级算法的基础

**局限性**：
- 字符集较大时空间开销大（O(m × |Σ|)）
- 模式串频繁变更时需要重建转移表
- 对于单次匹配场景，普通KMP的O(m)预处理更经济

理解KMP自动机，是从「会写KMP」到「真正理解字符串匹配本质」的重要一步。
