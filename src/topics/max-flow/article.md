# 最大流 (Max Flow)

## 概念解释

### 什么是流网络

流网络（Flow Network）是一个有向图 G = (V, E)，其中每条边 (u, v) 都有一个非负的**容量** c(u, v) >= 0。网络中有两个特殊的节点：

- **源点（Source, s）**：流的出发点，只有流出没有流入
- **汇点（Sink, t）**：流的终点，只有流入没有流出

除了源点和汇点，中间节点的流入量必须等于流出量，这就是**流量守恒**条件。

### 容量与流量

- **容量 c(u, v)**：边 (u, v) 上允许通过的最大流量
- **流量 f(u, v)**：边 (u, v) 上实际通过的流量
- 流量必须满足两个约束：
  - **容量约束**：0 <= f(u, v) <= c(u, v)
  - **流量守恒**：对任意中间节点 v，流入 v 的流量之和 = 流出 v 的流量之和

### 残余图

残余图（Residual Graph）是基于当前流构造的辅助图，表示每条边上还能增加多少流量：

- **正向边**：如果 c(u, v) - f(u, v) > 0，则残余图中存在边 (u, v)，残余容量为 c(u, v) - f(u, v)
- **反向边**：如果 f(u, v) > 0，则残余图中存在边 (v, u)，残余容量为 f(u, v)

反向边允许算法"撤销"之前分配的流量，这是找到全局最优解的关键。

### 增广路径

增广路径（Augmenting Path）是残余图中从源点 s 到汇点 t 的一条路径。沿增广路径可以增加的流量等于路径上所有边的残余容量的最小值（瓶颈值）。

### 最大流最小割定理

**最大流最小割定理**是网络流理论中最核心的定理：

> 网络中从源点到汇点的最大流 = 所有割中容量最小的那个割的容量

**割（Cut）**是将顶点集 V 分成两个集合 S 和 T（s ∈ S, t ∈ T）的一种划分，割的容量是所有从 S 指向 T 的边的容量之和。

这个定理告诉我们：流的上界恰好等于切断网络所需的最小代价。

## 为什么重要

最大流问题是图论和运筹学中最基本的问题之一，具有广泛的应用价值：

- **网络优化**：计算机网络中的带宽分配、管道网络中的流量分配
- **二分图匹配**：将二分图最大匹配问题转化为最大流问题求解
- **图像分割**：在计算机视觉中，使用最小割进行图像前景/背景分割
- **航班调度**：航空公司使用网络流优化航班分配和机组排班
- **项目选择**：在约束条件下选择最优项目组合

最大流算法是许多复杂优化问题的基础构建块。

## 核心原理

### Ford-Fulkerson 方法

Ford-Fulkerson 是一种迭代方法，核心思想非常直观：

1. 初始化所有边的流量为 0
2. 在残余图中寻找一条从 s 到 t 的增广路径
3. 计算增广路径的瓶颈容量 delta = min(路径上各边的残余容量)
4. 沿增广路径更新残余图：
   - 对路径上的每条正向边 (u, v)：增加 delta 的流量
   - 对路径上的每条反向边 (v, u)：增加 delta 的残余容量
5. 重复步骤 2-4，直到不存在增广路径
6. 最大流 = 所有增广路径的 delta 之和

### Edmonds-Karp 算法

Edmonds-Karp 是 Ford-Fulkerson 的一个具体实现，关键改进是使用 **BFS**（而非 DFS）寻找增广路径：

- 每次选择**最短**的增广路径（边数最少）
- 保证时间复杂度为 **O(VE^2)**
- 不依赖于容量的具体数值，是多项式时间算法

### 算法示例

考虑一个简单网络：

```
    s --10--> A --10--> t
    |                   ^
    +----10--> B --10---+
```

1. **第一轮**：找到路径 s → A → t，瓶颈 = min(10, 10) = 10，总流量 = 10
2. **第二轮**：找到路径 s → B → t，瓶颈 = min(10, 10) = 10，总流量 = 20
3. **无更多增广路径**，最大流 = 20

更复杂的例子中，反向边会发挥作用：

```
    s --10--> A --7--> t
    |         ^
    +--10--> B +--7---+
```

1. **第一轮**：s → A → t，流量 7，总流量 = 7
2. **第二轮**：s → B → A → t（利用 A→B 的反向边），流量 3，总流量 = 10

## 可视化说明

可视化演示展示了以下内容：

- **流网络图**：节点和有向边，边上标注"容量/当前流量"
- **增广路径高亮**：每一步找到的增广路径用彩色标记
- **残余图更新**：展示每次增广后残余图的变化
- **当前流量值**：实时显示从源点流出的总流量

通过逐步观察算法执行过程，可以直观理解增广路径的寻找和残余图的更新机制。

## 常见错误

### 1. 忘记更新残余图的反向边

这是最常见的错误。当沿增广路径增加流量时，必须同时更新正向边的残余容量（减少）和反向边的残余容量（增加）。忘记反向边会导致算法无法找到需要"调整"流量的路径。

### 2. 残余图更新不正确

更新残余图时，必须正确计算：
- 正向边的新残余容量 = 原残余容量 - delta
- 反向边的新残余容量 = 原残余容量 + delta

### 3. 使用 DFS 导致无限循环

当容量为无理数时，Ford-Fulkerson 使用 DFS 可能不收敛。即使容量为整数，DFS 选择的增广路径可能很差，导致迭代次数过多。使用 BFS（Edmonds-Karp）可以避免这个问题。

### 4. 混淆流量和容量

流量是实际通过的量，容量是允许通过的最大量。在残余图中，边的残余容量 = 容量 - 流量（正向边）或 = 流量（反向边）。

### 5. 忽视流量守恒

更新流量时，必须确保中间节点的流入量 = 流出量。增广路径的操作天然保证了这一点，但手动实现时容易出错。

## 实际应用

### 网络带宽分配

在计算机网络中，路由器之间的链路有带宽限制。最大流算法可以计算从发送端到接收端的最大数据传输速率，帮助网络管理员优化路由配置。

### 二分图最大匹配

将二分图转化为流网络：源点连到左部所有节点（容量 1），右部所有节点连到汇点（容量 1），原二分图的边容量为 1。最大流 = 最大匹配数。

### 图像分割

在计算机视觉中，将像素视为节点，相邻像素之间的边表示相似度。通过求最小割，可以将图像分为前景和背景两个区域。这就是著名的 **Graph Cut** 算法。

### 航空公司调度

航空公司需要将航班分配给机组人员。可以建模为二分图匹配问题：航班和机组人员分别在二部图的两侧，可行的分配方案作为边。最大流找到最优分配。

### 项目选择问题

给定一组项目和依赖关系，每个项目有收益或成本。通过构建适当的流网络，最大流最小割可以帮助选择最优的项目组合。

## 总结

最大流问题是图论中最经典、应用最广泛的问题之一。Ford-Fulkerson 方法提供了直观的框架：反复寻找增广路径并更新残余图。Edmonds-Karp 通过使用 BFS 保证了 O(VE^2) 的多项式时间复杂度。最大流最小割定理揭示了流与割之间的深刻对偶关系。

掌握最大流算法不仅是解决网络优化问题的基础，更是理解二分图匹配、图像分割等高级应用的关键。反向边的设计是算法正确性的核心——它赋予了算法"反悔"的能力，使其能够跳出局部最优，找到全局最大流。
