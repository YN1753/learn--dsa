# 最小费用最大流 (Min-Cost Max-Flow)

## 概念解释

### 什么是费用流网络

在普通流网络的基础上，费用流网络为每条边增加了一个**费用（Cost）**属性。形式化地，费用流网络是一个有向图 G = (V, E)，其中每条边 (u, v) 具有两个属性：

- **容量 c(u, v)**：边 (u, v) 上允许通过的最大流量，c(u, v) >= 0
- **单位费用 w(u, v)**：每通过一单位流量需要支付的费用

网络中有两个特殊节点：**源点 s** 和 **汇点 t**。

### 最小费用最大流问题

最小费用最大流（Min-Cost Max-Flow, MCMF）问题要求：

1. 首先找到从源点到汇点的**最大流**
2. 在所有最大流方案中，选择**总费用最小**的方案

总费用的计算公式为：**总费用 = sum( f(u, v) * w(u, v) )**，其中 f(u, v) 是边 (u, v) 上的流量。

### 流量守恒

与普通最大流一样，费用流也必须满足流量守恒条件：对于任意中间节点 v（非源点非汇点），流入 v 的流量之和等于流出 v 的流量之和。

### 残余图中的费用

在残余图中，正向边和反向边的费用处理方式不同：

- **正向边 (u, v)**：残余容量 = c(u, v) - f(u, v)，费用 = w(u, v)
- **反向边 (v, u)**：残余容量 = f(u, v)，费用 = -w(u, v)

反向边的费用为负值，意味着"退流"可以回收费用，这赋予了算法调整和优化的能力。

## 为什么重要

最小费用最大流是网络优化中的核心问题，具有广泛的实际应用价值：

- **运输优化**：在物流网络中，不同的运输路线有不同的运输成本，需要在满足最大运输量的前提下最小化总运输成本
- **任务分配**：将 n 个任务分配给 n 个工人，每个工人完成不同任务的成本不同，需要找到总成本最小的最大匹配
- **调度问题**：在满足最大处理量的同时，最小化调度方案的总代价
- **资源分配**：在有限资源约束下，以最小成本完成最大量的资源分配
- **网络路由**：在通信网络中，选择费用最低的路径来实现最大数据吞吐量

最小费用最大流统一了最大流和最短路径问题，是组合优化领域的基础算法之一。

## 核心原理

### 连续最短路径算法 (Successive Shortest Path)

连续最短路径算法是求解 MCMF 最直观的方法，核心思想是：

1. 初始化所有边的流量为 0
2. 在残余图中找到从源点到汇点的**费用最短路径**（而非边数最短）
3. 沿该路径增广，增广量为路径上的瓶颈容量
4. 重复步骤 2-3，直到不存在从 s 到 t 的路径
5. 此时的流即为最小费用最大流

每一步都选择费用最低的增广路径，确保在增加流量的同时，总费用的增量最小。

### 使用 SPFA 寻找最短路径

由于残余图中存在负权边（反向边的费用为负），不能直接使用 Dijkstra 算法。需要使用 **SPFA（Shortest Path Faster Algorithm）** 或 **Bellman-Ford** 算法来处理负权边：

```python
def spfa(network, source, sink):
    dist = {v: infinity for v in network.nodes}
    parent = {v: None for v in network.nodes}
    in_queue = {v: False for v in network.nodes}
    dist[source] = 0

    queue = [source]
    in_queue[source] = True

    while queue:
        u = queue.pop(0)
        in_queue[u] = False
        for edge in residual_edges(u):
            v = edge.to
            if dist[u] + edge.cost < dist[v]:
                dist[v] = dist[u] + edge.cost
                parent[v] = (u, edge)
                if not in_queue[v]:
                    queue.append(v)
                    in_queue[v] = True

    return dist, parent
```

### 势函数优化：使用 Dijkstra

为了避免 SPFA 的最坏情况 O(VE) 复杂度，可以引入**势函数（Potential）**将所有边的费用转化为非负值，从而使用 Dijkstra 算法：

1. 维护势函数 h(v)，初始时 h(v) = 0
2. 每次 SPFA/Dijkstra 后，更新势函数：h(v) = h(v) + dist(v)
3. 将边 (u, v) 的费用调整为：w'(u, v) = w(u, v) + h(u) - h(v)
4. 调整后的费用保证非负，可以使用 Dijkstra

势函数的思想类似于 Johnson 算法中的重新赋权技术。

### 时间复杂度

- **朴素连续最短路径（SPFA）**：O(V * E * max_flow)，每次 SPFA 为 O(VE)，最多增广 max_flow 次
- **势函数优化（Dijkstra）**：O(V * max_flow * log V)，每次 Dijkstra 为 O(E log V)
- **使用最小费用流的特殊性质**：可以通过更精细的分析得到更好的上界

### 算法示例

考虑一个简单的费用流网络：

```
节点: s, A, B, t
边: s→A (容量=2, 费用=1), s→B (容量=2, 费用=3),
    A→B (容量=1, 费用=1), A→t (容量=2, 费用=2),
    B→t (容量=2, 费用=1)
```

**第 1 轮**：SPFA 找到最短路径 s → A → t，费用 = 1 + 2 = 3，瓶颈 = 2，总流量 = 2，总费用 = 6

**第 2 轮**：SPFA 找到最短路径 s → B → t，费用 = 3 + 1 = 4，瓶颈 = 2，总流量 = 4，总费用 = 14

**结果**：最大流 = 4，最小费用 = 14

### 为什么贪心策略是正确的

连续最短路径算法采用贪心策略：每轮选择费用最短路径增广。为什么这样能得到全局最优？

关键在于**反向边的负费用**。当算法在后续轮次发现更优的路径时，可以通过反向边"退流"并回收之前支付的费用。这使得算法能够调整之前的决策，而不仅仅是做不可撤销的选择。

形式化地说，每次增广都沿着残余图中的最短路径进行，这保证了在当前流量下费用增量最小。当流量达到最大时，总费用也是最小的。这个正确性基于网络流的对偶理论和互补松弛条件。

### 与其他算法的关系

最小费用最大流是多个经典问题的推广：

- **最大流问题**：当所有边的费用为 0 时，MCMF 退化为普通最大流
- **最短路径问题**：当网络只有一条从 s 到 t 的路径且容量为 1 时，MCMF 退化为最短路径
- **二分图最大权匹配**：可以转化为 MCMF 求解
- **运输问题**：经典的线性规划问题，可以用 MCMF 高效求解

### SPFA 算法详解

SPFA（Shortest Path Faster Algorithm）是 Bellman-Ford 算法的队列优化版本，专门用于求解含负权边的单源最短路径：

1. 初始化 dist[s] = 0，其他节点 dist[v] = 无穷大
2. 将源点 s 加入队列
3. 取出队首节点 u，对 u 的所有邻接边 (u, v) 进行松弛操作
4. 如果 dist[u] + w(u, v) < dist[v]，更新 dist[v]，若 v 不在队列中则加入
5. 重复步骤 3-4 直到队列为空
6. 如果某个节点入队次数超过 V 次，说明存在负环（在 MCMF 中不会出现）

SPFA 的平均时间复杂度为 O(kE)，其中 k 是一个较小的常数（通常 k <= 2）。但在最坏情况下，SPFA 的复杂度为 O(VE)，与 Bellman-Ford 相同。

### 势函数优化详解

势函数（Potential Function）优化是 MCMF 的重要加速技术。核心思想是通过重新赋权将负权边转化为非负权边：

**初始化**：h(v) = 0 对所有节点 v

**每次最短路径后**：
1. 更新势函数：h(v) = h(v) + dist(v)（dist 为本轮 SPFA 计算的距离）
2. 重新赋权：对每条残余边 (u, v)，新费用 w'(u, v) = w(u, v) + h(u) - h(v)

**关键性质**：
- 重新赋权后的费用 w'(u, v) >= 0（由三角不等式保证）
- 最短路径不变（因为从 s 到任意节点 v 的路径费用偏移量相同）
- 因此可以使用 Dijkstra 代替 SPFA，每次最短路径复杂度从 O(VE) 降至 O(E log V)

### MCMF 的 Primal-Dual 算法

Primal-Dual 算法是 MCMF 的另一种实现方式，基于线性规划的对偶理论：

1. 使用 Bellman-Ford 计算初始势函数
2. 使用 Dijkstra 在重新赋权后的图中找最短路径
3. 沿最短路径增广
4. 更新势函数
5. 重复直到无增广路径

Primal-Dual 算法的时间复杂度为 O(V^2 * E * log V) 或更优，是实际应用中最常用的高效算法之一。

## 常见错误

### 1. 忘记处理残余图中的负费用边

这是最常见的错误。当正向边有流量时，残余图中的反向边具有负费用。如果使用 Dijkstra 而不加势函数处理，会得到错误结果。必须使用 SPFA/Bellman-Ford，或者引入势函数后才能使用 Dijkstra。

### 2. 费用更新不正确

增广时，总费用应增加：**delta * 路径上的总费用**，而不是仅增加路径费用。每通过一单位流量就要支付一单位费用，所以总费用增量 = 流量增量 × 路径费用。

### 3. 混淆最大流和最小费用的优先级

MCMF 首先保证流量最大，在流量相同的方案中选择费用最小的。不能为了降低费用而牺牲流量。

### 4. 势函数更新错误

使用势函数优化时，必须在每次最短路径计算后正确更新势函数：h(v) += dist(v)。忘记更新或更新方式错误会导致边权变为负数，Dijkstra 结果出错。

### 5. 反向边费用符号搞错

正向边的费用为 w(u, v)，反向边的费用为 -w(u, v)。如果符号搞反，算法会往费用更高的方向搜索，得到最大费用流而非最小费用流。

## 实际应用

### 物流运输

物流公司需要将货物从仓库运送到多个目的地。不同路线的运输成本不同，每条路线有运力限制。MCMF 可以在满足最大运输量的同时，找到总运输成本最低的方案。

### 任务分配

经典的二分图带权匹配问题：n 个工人和 n 个任务，每个工人完成不同任务有不同的成本。将问题建模为费用流网络，MCMF 找到总成本最小的完美匹配，这就是**匈牙利算法**的费用流实现。

### 网络路由

在计算机网络中，不同的路由路径有不同的延迟或带宽费用。MCMF 帮助找到在满足最大吞吐量的同时，总延迟或总费用最低的路由方案。

### 航班调度

航空公司需要安排航班并分配飞机。每条航线有容量限制和运营成本，MCMF 可以优化航班分配，在最大化航班数量的同时最小化总运营成本。

### 生产调度

工厂有多种产品需要生产，不同生产线有不同的产能和成本。MCMF 帮助制定生产计划，在满足最大产量的同时最小化生产成本。

## 总结

最小费用最大流是网络流理论中最重要的扩展之一，它将最大流和最短路径问题统一在一个框架中。连续最短路径算法通过反复寻找残余图中的费用最短路径来逐步增广，每次选择费用增量最小的路径，从而保证最终得到费用最小的最大流方案。

SPFA/Bellman-Ford 处理残余图中的负权边，势函数优化则通过 Johnson 式重新赋权将问题转化为非负权最短路径，使 Dijkstra 得以应用。掌握 MCMF 不仅是解决复杂网络优化问题的基础，更是理解二分图带权匹配、任务调度等高级应用的关键。

反向边的负费用机制是算法正确性的核心——它允许算法在发现更优方案时"退回"已分配的流量，从而跳出局部最优，找到全局最小费用最大流。
